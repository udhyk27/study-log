# 리눅스 기본 명령어 및 와일드카드
1. 와일드카드 기호
*: 모든 문자를 의미함 (예: ls *.txt는 확장자가 txt인 모든 파일 출력)
?: 딱 하나의 문자에 대응할 때 사용함

2. 목록 및 경로 확인 (ls, pwd)
ls -a: 숨겨진 파일을 포함하여 모든 파일 목록 출력
ls -l: 파일 종류, 권한, 사이즈 등 상세 정보 표시
ls -R: 하위 디렉터리 내용까지 모두 출력
pwd: 현재 내가 위치한 디렉터리의 전체 경로 출력

3. 디렉터리 생성 및 이동 (mkdir, cd)
mkdir: 새로운 디렉터리 생성
mkdir -p: 존재하지 않는 상위 디렉터리까지 한 번에 생성함
cd ..: 상위 디렉터리로 이동
cd ~: 사용자의 홈 디렉터리로 이동

# 시스템 및 프로세스 관리
1. 시스템 종료 및 정보
shutdown, reboot, halt: 시스템 종료 및 재부팅 관련 명령어임
hostname: 현재 시스템의 호스트 이름 확인 및 설정
uname -a: 커널 버전 및 시스템 정보 상세 출력

2. 프로세스 제어
ps: 현재 실행 중인 프로세스 정보 확인
top: 실시간 CPU 및 메모리 사용량 모니터링
kill: PID를 이용하여 프로세스 종료
jobs: 백그라운드 작업 목록 확인
bg / fg: 작업을 백그라운드나 포어그라운드로 전환함

# 파일 편집 및 텍스트 처리
1. 파일 내용 확인
cat: 파일 내용을 화면에 출력하거나 여러 파일을 합침
head / tail: 파일의 앞부분이나 뒷부분 일부만 출력함
more / less: 화면 단위로 끊어서 파일 내용을 보여줌

2. 텍스트 필터 및 편집
grep: 특정 문자열이나 패턴을 검색함
sed: 스트림 에디터로 텍스트 치환 및 삭제 수행
awk: 텍스트 패턴 처리 및 리포트 생성에 강력함
cut: 파일의 특정 필드(열)만 추출함
sort: 줄 단위 정렬 및 검색 수행

# 권한 및 소유권 (자주 출제)
chmod: 파일이나 디렉터리의 허가권(rwx) 변경
chown: 소유자 및 그룹 소유권 변경
chgrp: 그룹 소유권만 변경
umask: 파일 생성 시 기본 권한 마스크 설정

# 압축 및 아카이브
tar: 파일들을 하나로 묶거나(c) 품(x). tvf로 내용 확인 가능
gzip / gunzip: .gz 파일 압축 및 해제
bzip2 / bunzip2: .bz2 파일 압축 및 해제
xz / unxz: 가장 압축률이 좋은 .xz 관련 유틸리티임

# 네트워크 관련
ifconfig: 랜카드 주소 할당 및 상태 확인 (최신은 ip 명령 사용)
netstat: 네트워크 연결 상태 및 라우팅 테이블 확인
ping: 네트워크 연결 여부 테스트
arp: IP 주소와 MAC 주소 매핑 정보 확인
nslookup / dig: DNS 서버에 도메인 정보 질의

** ls -l 실행 시 나오는 첫 번째 필드(-, d, l)를 통해 파일 종류를 구분하는 문제가 자주 나옴
** mkdir -p 옵션은 하위 경로까지 한 번에 생성한다는 점이 실기/필기 공통 빈출 포인트임
** grep은 특정 단어를 찾는 용도, find는 파일 자체를 찾는 용도임을 명확히 구분해야 함
** ps는 현재 시점의 정보를 보여주고 top은 실시간(지속적) 정보를 보여준다는 차이가 있음
** tar 사용 시 z(gzip), j(bzip2), J(xz) 옵션과 확장자를 바르게 매칭하는 것이 중요함

# RAID 핵심 요약
RAID 관련 주요 기술
스트라이핑: 데이터를 여러 디스크에 나누어 저장하여 속도를 높임 (RAID 0)
미러링: 데이터를 여러 디스크에 똑같이 복제하여 저장함 (RAID 1)
패리티: 데이터 복구를 위해 추가로 저장하는 비트 (RAID 5, RAID 6)

결함 허용 (오류 발생 시 복구 가능 여부)
복구 가능: RAID 1, RAID 5, RAID 6, RAID 10
복구 불가: RAID 0 (하나라도 고장 나면 전체 데이터 복구 불가)
RAID 레벨별 실제 사용 용량 (N = 전체 디스크 개수)
RAID 0: 100% 그대로 모두 사용 가능 (예: 10GB 디스크 4개 = 40GB 사용)
RAID 1: 전체 용량의 50%만 사용 가능 (절반은 미러링 복제에 사용)
RAID 5: N-1개 용량 사용 가능 (디스크 1개 용량은 패리티로 사용)

** 스페어(예비) 디스크가 포함된 응용 계산 문제
문제에 스페어 디스크가 언급되면 전체 개수에서 먼저 뺍니다.
예: 디스크 8개 중 1개를 스페어로 구성하면 남은 것은 7개. 이를 RAID 5로 구성하면 패리티 1개를 추가로 빼서 최종 6개 분량만 사용 가능.
** 패리티 공간 비율 계산
RAID 5를 디스크 3개로 구성할 경우, 1개가 패리티이므로 전체 공간의 33.3%가 패리티 공간으로 사용됨
** RAID 6의 가용 용량은 전체 개수에서 2개를 빼는 N-2 공식을 적용함

# LVM
1. LVM 구성 및 특징
구성 순서: PV -> VG -> LV
유연성: 물리 디스크 1개로 여러 파티션을 만들거나, 2개를 1개로 묶을 수 있음
용량 관리: 용량 증설이 매우 쉬움. 확장 시 데이터 손실은 없으나, 축소 시에는 손실이 발생함

2. LVM 주요 단위
LV: 사용자가 필요한 만큼 할당받아 사용하는 가상 파티션
PE: LVM을 구성하는 최소 단위 블록임

** LVM 생성 순서는 무조건 PV -> VG -> LV
** LVM 확장은 안전하지만, 파티션 축소 시에는 데이터 손실 위험이 있다

# 패키지 및 아카이브 관리
rpm 및 yum 명령어
rpm 삭제: 패키지 삭제 시 의존성을 무시하려면 -e --nodeps 옵션을 씀 
rpm 정보: -qa(전체 설치 패키지), -qi(상세 정보), -qf(원하는 형태로 출력)

2. tar 압축 및 옵션

주요 옵션: c(생성), x(해제), t(목록 확인), r(추가 묶기) 
tvf 묶음: 묶인 파일의 내용을 상세하게 확인할 때 씀 

3. 소스 파일 설치 과정
설치 단계: configure -> make -> make install 순서대로 진행함

makefile생성 / 컴파일 / 설치

** 프린터 출력 시 BSD 계열(lpr)과 System V 계열(lp) 명령어 짝짓기 문제에 주의 
** rpm 패키지 삭제 시 발생하는 의존성 오류를 무시하는 --nodeps 옵션은 단골 문제
** tar 압축 확장자(J, j, z) 구분 및 압축률이 가장 좋은 것(xz)을 묻는 유형 대비 필요 
** 소스 코드 설치 순서는 무조건 configure -> make -> make install임

$: 줄의 끝 부분 

특수문자 → \(백슬래시) 아래 단어 시작끝과 같이 씀
<: 단어의 시작 부분 
>: 단어의 끝 부분

치환 예시 2: :% s/\<linux/Linux/g -> 단어의 시작이 linux인 것을 모두 대문자 Linux로 바꿈 

** 레드햇 계열(rpm, yum)과 데비안 계열(dpkg, apt-get)의 기본 패키지 관리 도구 짝짓기가 빈출됨

** 정규표현식 치환 문제에서 ^(줄 시작), $(줄 끝), <(단어 시작), >(단어 끝) 기호의 위치와 의미를 정확히 묻는 문제가 나옴

# 프로세스 관리
1. 프로세스 종료 (kill vs killall)
kill: 특정 프로세스의 PID나 작업 번호(%번호)를 인자로 사용하여 종료함 (예: kill 513 514, kill %2)
killall: 프로세스 이름을 인자로 사용하여 연관된 모든 프로세스를 한꺼번에 종료함

2. 인자값에 따른 명령어 분류
PID(숫자)를 사용하는 명령어: kill, renice
프로세스명(문자)을 사용하는 명령어: killall, nice

** 편집기와 개발자 짝짓기(vi-빌 조이, emacs-리처드 스톨만, vim-브람 무레나르) 문제가 단골로 나옴
** X윈도 환경에서만 실행되는 편집기(kwrite, gedit)를 고르는 문제가 자주 출제됨
** 명령어 실행 시 인자값으로 PID를 주는지(kill, renice), 프로세스명을 주는지(killall, nice) 묻는 문제가 핵심임

# ps 명령어
1. ps 명령 개요
실행 중인 프로세스의 상태 정보(PID, TTY, TIME, CMD)를 출력함
백그라운드에서 실행 중인 데몬을 확인하거나, 로그인 후 현재 이용 중인 셸을 확인할 때 사용함

2. 주요 옵션 및 상태 코드
Z: 작업이 종료되었으나 완전히 회수되지 않아 메모리를 계속 차지하고 있는 상태(좀비 프로세스)임
l: 프로세스의 우선순위(NI) 값을 확인할 때 씀
aux: 시스템에서 실행 중인 모든 프로세스를 자세히 출력함

# 프로세스 우선순위 및 제어
1. NI (Nice) 값
설정 범위는 -20부터 19까지임
숫자가 작을수록(마이너스로 갈수록) 우선순위가 높음

2. nice vs renice 비교
nice: 프로세스명(문자)을 인자로 받아 새로운 프로세스를 실행할 때 우선순위를 지정함
renice: 기존에 실행 중인 프로세스의 PID(숫자)를 인자로 받아 우선순위를 변경함

3. 명령어 사용 예시
nice -10 bash: 새로운 bash 프로세스를 실행하며 NI 값을 10으로 설정함
renice 10 1222: 이미 실행 중인 PID 1222번 프로세스의 NI 값을 10으로 변경함
nice bash: NI 값을 따로 지정하지 않으면 기본값인 10이 적용되어 기본 상태(0)보다 우선순위가 낮아짐

** 로그인한 이후에 현재 자신이 사용하고 있는 셸을 확인할 때 쓰는 명령어로 ps가 단골 출제됨
** Z 상태는 프로세스가 종료되었음에도 시스템 자원(메모리)을 차지하고 있는 불량 상태임을 기억해야 함

# 데몬 (Daemon) 프로세스
1. 실행 방식 비교
standalone: 메모리에 항상 독자적으로 상주하여 속도가 빠름
inetd: 서비스 요청이 있을 때만 메모리에 올라와서 관리에 효율적임
데몬은 주기적이고 지속적인 요청 처리를 위해 백그라운드에서 실행되는 프로세스임

# 주요 시그널 번호
1. kill 명령어와 시그널
kill 명령어에 PID를 써서 특정 시그널을 보냄
1 (SIGHUP): 터미널과 연결이 끊어질 때 발생함
2 (SIGINT): 실행 중단 (종료) 시그널이며, <ctrl+c>와 같음
3 (SIGQUIT): 종료 및 코어 덤프를 생성하며, <ctrl+>와 같음
9 (SIGKILL): 프로세스 즉시 강제 종료 (무조건 종료) 시그널임
15 (SIGTERM): 정상 종료를 요구하며 kill 명령어의 기본 시그널임
20 (SIGSTOP): 작업 일시 정지 (대기) 시그널이며, <ctrl+z>와 같음

# 프로세스 호출 및 초기화
1. 프로세스 생성 방법
fork: 기존 프로세스를 그대로 복제하여 새로운 프로세스를 생성함.
exec: 원래의 프로세스를 새로운 프로세스로 완전히 대체함.

2. 최상위 프로세스 (PID 1번)
CentOS 6: init 프로세스를 사용했음.
CentOS 7: systemd 프로세스를 발생시킴.

** 백그라운드에서 실행 중인 프로세스를 확인하는 명령어는 jobs임이 자주 출제됨
** 데몬 실행 방식 중 메모리 효율이 좋은 것은 inetd, 항상 상주하는 것은 standalone임을 묻는 비교 문제가 단골임
** 프로세스를 강제로 즉시 종료하는 시그널 번호는 9번(SIGKILL)이고, 정상 종료는 15번(SIGTERM)임을 정확히 구분해야 함
** 기존 프로세스를 대체하는 방식은 exec, 복제하는 것은 fork로 구분 지어 묻는 지문이 나옴

# 설정 파일 및 셸 변수
1. 전역 파일과 지역 파일
전역 설정: /etc 디렉터리에 위치함
지역 설정: ~/ (사용자 홈 디렉터리)에 위치함
bash_profile: 환경 변수를 설정함
bashrc: alias(별칭) 및 함수를 설정함
profile: 시작 관련 프로그램을 설정함

2. 변수 확인 및 설정 명령어
printenv, env: 환경 변수만 출력함
set: 환경 변수와 셸 변수를 모두 포함하여 출력 및 설정함
unset: 설정된 셸 변수나 환경 변수를 제거(해제)함
chsh: 사용할 셸을 변경함

3. 히스토리(명령어 기록) 관련 기호

!0: 히스토리에서 0번째 명령어를 실행함
!?al: 최근 실행한 명령 중 'al' 문자열이 포함된 명령을 찾아 실행함
echo $USER: 현재 로그인한 사용자 이름을 출력함

# 시스템 주요 정보 파일
1. 사용자 및 셸 정보 파일
/etc/passwd: 로그인 시 할당되는 기본 셸 정보가 저장됨 (확인 명령어: echo $SHELL)
/etc/shells: 시스템에서 현재 사용 가능한 셸의 목록이 들어있음 (현재 사용 중인 셸 확인 명령어: ps)

2. 마운트 관련 정보 파일
/etc/mtab: 현재 마운트되어 있는 정보를 저장함
/etc/mounts: 실시간 마운트 정보를 보여줌
/etc/fstab: 부팅 시 자동으로 마운트할 장치 정보를 지정해 두는 파일임

3. /etc/fstab 파일의 6개 필드 구조
1번째 필드: UUID, LABEL, 장치 파일명 (마운트 포인트는 쓸 수 없음)
2번째 필드: 마운트 경로 (마운트 포인트)
3번째 필드: 파일 시스템 유형
4번째 필드: 마운트 옵션 (예: noexec 등. 단, userquota는 여기서 설정하지 않음)
5번째 필드: 덤프(백업) 시 포함 여부
6번째 필드: 부팅 시 파일 시스템 점검(fsck) 순서

** 부팅 시 자동 마운트 정보를 담은 /etc/fstab 파일의 6가지 필드 순서와 의미(특히 1번 필드에 장치명, 2번 필드에 마운트 포인트가 온다는 점)를 묻는 문제가 매번 출제됨
** 환경 변수만 출력하는 명령어(env)와 셸 변수까지 모두 출력하는 명령어(set)의 차이를 정확히 구분해야 함
** 사용자의 기본 셸 정보가 담긴 파일(/etc/passwd)과 사용 가능한 전체 셸 목록 파일(/etc/shells)을 헷갈리지 않도록 주의해야 함
** 직전 명령어를 재실행하는 !! 기호와 특정 문자열을 포함한 명령을 찾는 !?문자열 형태의 명령어 재실행 방법이 자주 나옴

# 셸(Shell) 관리 및 특징
1. chsh 명령어 옵션
l: 사용 가능한 셸의 목록 정보를 확인함
s: 지정한 셸로 변경하며 절대 경로를 함께 입력해야 함 (예: chsh -s /bin/csh)
u: 도움말(help)을 출력함

2. 주요 셸 종류 및 특징
발전 순서: sh -> csh -> tcsh -> bash 순서임
bourne shell (sh): 가장 오래된 기본 셸임
csh: 버클리 대학의 빌 조이가 개발했으며, 히스토리, 별칭(alias), 작업 제어 기능이 유용함
tcsh: csh 기능에 명령행 완성 기능이 통합된 셸임
bash: GNU 프로젝트로 개발되었으며, 다양한 운영체제에서 널리 쓰임
dash: /bin/sh를 가장 작게 구현한 셸로 POSIX 표준과 호환됨

# 디스크 및 파일 시스템 관리
1. 디스크 용량 확인
du: 디렉터리나 사용자별 디스크 사용량(use)을 확인함
df: 파티션별 마운트된 디스크의 남은 용량(avail)을 확인함

2. fdisk 및 파일 시스템 생성
fdisk 실행 모드 옵션: w(저장 후 종료), q(저장 없이 종료), d(파티션 삭제), l(파티션 테이블 확인)임
mkfs.xfs: 파티션에 xfs 파일 시스템을 생성함

3. 장치 정보 확인 명령어
blkid: 파티션 생성 시 부여되는 고유 식별자인 UUID와 파일 시스템 정보를 확인함
lsblk: 연결된 저장 장치의 목록과 이동식 장치 여부를 확인함

4. 디스크 쿼터(Quota) 관리
repquota: 파일 시스템의 쿼터 상태(soft, hard limit)를 요약해서 확인함
edquota: 사용자나 그룹의 디스크 쿼터를 설정함
quota: 개별 사용자에 설정된 쿼터 정보를 확인함
xfs_quota: xfs 파일 시스템 전용 디스크 쿼터를 설정하고 관리함

** chsh -s 명령어로 셸을 변경할 때는 반드시 /bin/csh처럼 절대 경로를 입력해야 하는 점이 자주 출제됨
** 셸 특징 연결 문제에서 빌 조이+히스토리 기능은 csh, GNU 프로젝트는 bash임
** 디스크의 남은 공간 전체를 볼 때는 df, 특정 폴더나 파일이 차지하는 용량을 볼 때는 du를 쓴다는 차이점을 묻는 문제가 나옴
** 파티션의 식별자인 UUID를 확인하는 명령어는 무조건 blkid임 (UUID = blkid 세트로 암기)
** 쿼터를 실제로 '설정'하는 명령어는 edquota이고, 전체 상태를 요약해서 '보고'하는 명령어는 repquota임을 헷갈리지 않게 주의해야 함

# 권한(Permission) 및 소유권 관리
1. umask 및 허가권 계산 
파일의 기본 권한은 666임
파일 생성 시 기본 권한에서 umask 값을 빼서 최종 권한을 구함
예: umask가 0022면 파일 허가권은 666 - 22 = 644 (rw-r--r--)가 됨

2. 파일 허가권 (chmod) 
r(읽기=4), w(쓰기=2), x(실행=1) 기호와 숫자를 사용함
파일 유형(첫 번째 필드): -는 파일, d는 디렉터리, l은 심볼릭 링크임
권한 부여 대상: u(소유자), g(그룹), o(타인)임
예: chmod g+r lin.txt (그룹에 읽기 권한 추가), chmod g=r lin.txt (그룹 권한을 읽기로만 설정)
-R 옵션: 하위 디렉터리와 파일까지 모두 포함하여 권한을 변경함

3. 특수 권한 (특수 비트) 
SetUID (4): 소유자 권한으로 실행됨
SetGID (2): 그룹 권한으로 실행됨
Sticky Bit (1): 본인이 생성한 파일만 삭제할 수 있게 설정함
주의: Sticky Bit는 디렉터리에만 설정 가능하며 파일에는 설정할 수 없음 (예: 파일에 chmod g+t /project 설정 불가)
적용 예: Sticky Bit(1) + SetGID(2) = 3 -> 맨 앞에 붙여 chmod 3770 /project로 설정함

4. 소유권 관련 명령어 
chown: 파일 소유자와 그룹을 같이 변경할 수 있음
chgrp: 그룹만 변경할 수 있음
ls: 파일 및 디렉터리의 소유자를 확인하거나, 특수 권한(SetUID 등) 적용 여부를 확인할 때 씀

** Sticky Bit는 파일이 아닌 디렉터리에만 적용된다는 점을 이용한 낚시성 지문을 조심해야 함

** 실시간 시스템 상태 및 CPU 사용량 모니터링은 top, 터미널 종료 후에도 작업 유지는 nohup임
** cmake로 설치하는 프로그램(MySQL, KDE, LMMS)과 아닌 것(PHP)을 구별하는 문제가 나옴

# 파일 시스템 및 자원 공유
1. 파일 시스템 점검 및 마운트
파일 시스템 점검: ext4는 fsck를 사용하고, xfs는 xfs_repair 명령을 사용함 
마운트 명령어 구조: mount -t 파일시스템유형 장치명 마운트포인트 형식으로 작성함

저널링: 시스템 오류 시 복구를 돕는 기능으로 ext3부터 적용되었으며 ext4, xfs도 지원함 

2. 공유 및 보안 관련 파일
/usr/bin/passwd: 일반 사용자도 비밀번호를 변경할 수 있도록 Set-UID 같은 특수 권한이 설정되어 있는 대표적 파일임 
/etc/shadow: 암호화된 비밀번호가 담겨있어 일반 사용자는 절대 내용을 볼 수 없음 
CIFS / SAMBA: 윈도우와 리눅스(유닉스) 환경 사이에서 자원을 공유하게 해주는 시스템임 
NFS: 한 리눅스의 디렉터리를 네트워크상의 다른 사용자들이 손쉽게 접근하고 공유하도록 해줌 

** 파일 시스템별 점검 명령어가 다르다는 점(ext4는 fsck, xfs는 xfs_repair)을 묻는 문제가 단골임 
** 이종 OS 간(윈도우와 리눅스) 공유는 SAMBA(CIFS), 같은 리눅스 간 자원 공유는 NFS라는 차이를 명확히 구분해야 함 
** CUPS는 애플이 만든 프린팅 시스템이고 IPP는 원격 인쇄용 프로토콜임을 묻는 개념 매칭 문제가 나옴 
** 마운트 정보를 수정하거나 갱신할 때 쓰는 -o remount 옵션과 시스템 기본 프롬프트를 바꾸는 PS1 변수의 역할이 중요함

# 네트워크 설정 파일 및 명령어
1. 주요 네트워크 설정 파일
/etc/resolv.conf: 사용할 DNS 서버 주소(nameserver)를 지정함 
/etc/hosts: IP 주소와 호스트명을 매핑하여 로컬에서 DNS 기능을 수행함 
/etc/services: 서비스별 포트 번호를 확인함 
/etc/protocols: 시스템에서 사용 가능한 프로토콜을 확인함 
/etc/sysconfig/network: 네트워크 사용 유무나 게이트웨이 주소 등 기본 정보를 설정함 
/etc/sysconfig/network-scripts: 개별 네트워크 인터페이스의 환경 설정을 저장함 

2. 네트워크 상태 확인 명령어
mii-tool: 네트워크 인터페이스의 물리적 연결 상태를 확인함 
ethtool: 이더넷 카드의 상세 설정 및 물리적 케이블 상태를 확인함 
netstat: 네트워크 연결 상태를 보여주며 SYN Flooding 공격을 탐지할 수 있음 
netstat 옵션: -r 옵션을 쓰면 라우팅 테이블과 게이트웨이 확인도 가능함 
route: 라우팅 테이블과 게이트웨이 주소를 확인함 
ip: 라우팅 테이블 및 네트워크 정보를 확인함 

** 클라우드 서비스 개념과 제공하는 자원(인프라, 플랫폼, 소프트웨어)을 짝짓는 문제가 자주 출제됨
** 가상화 기술 중 전가상화(KVM)와 반가상화(Xen)를 구별하는 문제가 핵심임
** 시스템 장애 대비용은 고가용성(HA) 클러스터이고 성능 향상용은 베어울프(HPC) 클러스터임을 명확히 구분해야 함 
** DNS 서버 주소를 직접 입력하는 파일은 /etc/resolv.conf 이고 IP와 도메인을 매핑하는 파일은 /etc/hosts 임을 묻는 문제가 단골임 
** 물리적인 랜선 연결 상태를 확인하는 명령어는 mii-tool 과 ethtool 임

# 네트워크 IP 및 서브넷
1. IP 클래스 대역
A클래스: 127.0.0.1은 자기 자신을 가리키는 루프백(Loopback) 주소임
B클래스: 128~ 부터 시작함
C클래스: 192~ 부터 시작하며, 사설 IP 네트워크는 256개임
D클래스: 224~ 부터 시작함

2. 서브넷 및 브로드캐스트 계산 (예: /26)
호스트 개수: 32비트 중 26비트를 네트워크로 쓰면 6비트가 남으므로 2^6 = 64개임
서브넷 범위: 64개 단위로 대역이 나뉨 (0~63, 64~127, 128~191 등)
주소 할당 규칙: 대역의 첫 번째(예: 128)는 네트워크 주소, 마지막(예: 191)은 브로드캐스트 주소로 씀
게이트웨이 및 호스트: 첫 번째와 마지막을 제외한 나머지 주소(예: 129~190)를 할당함

# 주요 네트워크 서비스
1. FTP (파일 전송)
포트 역할: 20번 포트는 데이터 전송, 21번 포트는 제어용임
주요 명령어: 업로드(put), 다운로드(get, recv), 진행 상태 표시(hash, # 기호로 출력)
특징: Active/Passive 모드가 있고 익명 계정 접속이 가능하며 FTP 서버 설치가 필수임

2. 원격 접속 (X 윈도, SSH, Telnet)
X 윈도 원격: 보내는 쪽(클라이언트)은 DISPLAY 환경 변수 변경, 받는 쪽(서버)은 xhost로 접근 제어함
SSH 및 Telnet: -l 옵션 뒤에 계정명을 넣어 원격지에 접속함 (예: ssh -l 계정명 IP주소)
Telnet 서버: 서버로 쓰려면 telnet-server 패키지를 설치해야 함

3. 메일 및 자원 공유
전자우편: SMTP는 송신 및 메일 교환용, POP3와 IMAP은 수신 전용 프로토콜임
SAMBA: 리눅스와 윈도우 운영체제 간의 파일 및 자원 공유를 지원하며 SMB/CIFS 프로토콜을 사용함

# 주요 통신 국제기구
1. 기구별 핵심 역할
ICANN: IP 주소와 도메인(DNS)을 총괄함 (초기 IANA에서 이관됨)
IEEE: LAN 표준 규격을 제정함
ISO: OSI 7계층 모델을 제정함
EIA: 전자산업협회로 RS-232, LAN 케이블 규격을 다룸

** 서브넷 영역이 주어졌을 때 가장 첫 주소는 네트워크 주소, 가장 끝 주소는 브로드캐스트 주소라는 점을 묻는 계산 문제가 필수임
** FTP 프로토콜에서 20번(데이터)과 21번(제어) 포트의 역할을 반대로 섞어 놓는 오답 지문을 조심해야 함
** 메일 서버 관련 문제에서 보내는 역할(SMTP)과 받는 역할(POP3, IMAP)을 정확히 구분해야 함
** 리눅스와 윈도우(이기종) 간 공유는 무조건 SAMBA(SMB/CIFS)임
** 단체 연결 문제에서 도메인/IP는 ICANN, LAN 표준(알파벳 E가 많음)은 IEEE로 암기해 두면 편함

# OSI 7계층 요약
1. 계층별 특징 및 장비
1계층 (물리): 비트(bit) 단위 전송. 허브(Hub), 리피터(Repeater) 장치를 사용함
2계층 (데이터링크): 프레임(frame) 단위 전송. 브리지(Bridge), 스위치(Switch) 장치를 사용함
3계층 (네트워크): 패킷(packet) 단위 전송. 통신 경로 선택(라우팅)을 수행함. IP, ICMP, IGMP, BGP 프로토콜과 라우터(Router) 장치를 사용함
4계층 (전송): 세그먼트(segment) 단위 전송. TCP, UDP 프로토콜과 게이트웨이(Gateway) 장치를 사용하며 가장 많은 기능을 지원함
5계층 (세션): 데이터 단위는 데이터(data). 동기화 유지 및 데이터 전송 순서를 관리함
6계층 (표현): 데이터 단위는 데이터(data). 번역, 압축, 암호화 기능을 수행함
7계층 (응용): 데이터 단위는 데이터(data). 사용자에게 네트워크 서비스를 제공함

** 1계층부터 4계층까지의 데이터 단위(bit -> frame -> packet -> segment) 순서를 묻는 문제가 자주 출제됨
** 각 계층별 대표 장비 매칭(스위치는 2계층, 라우터는 3계층, 게이트웨이는 4계층)이 핵심 포인트임
** ICMP, IGMP, BGP 등 주요 프로토콜이 3계층(네트워크)에 속한다는 점을 기억해야 함
** 세션 계층은 '동기화', 표현 계층은 '압축/암호화'라는 키워드로 구분하여 암기함


# 네트워크 및 X 윈도 시스템 요약
1. 네트워크 프로토콜 및 멀티미디어
ARP: IP 주소를 물리적인 MAC 주소로 매칭하거나 확인하는 프로토콜임
이미지 프로그램: EOG(이미지 뷰어), Gimp(사진 편집), ImageMagicK(이미지 변환 및 편집)
동영상 프로그램: Totem(기본 동영상 재생기)

2. X 윈도 기본 구조 및 통신
X 윈도 통신: TCP 6000번 포트 및 X 프로토콜을 사용함
X 클라이언트: X 윈도에서 동작하는 실제 응용 프로그램임
클라이언트/서버 구조: 프로그램(클라이언트)이 화면 출력(서버)을 요청하는 방식임

3. X 윈도 환경 변수 및 설정 파일
DISPLAY 변수: X 클라이언트를 원격지로 전송할 때 변경함 (예: :0은 로컬 시스템의 첫 번째 X 서버를 의미함)
.Xauthority: 인증 키(Cookie) 정보가 저장되는 파일임
xauth vs xhost: xauth는 키(Cookie) 값을 이용하고, xhost는 호스트명이나 IP 주소를 이용하여 접근을 제어함

4. X 윈도 명령어 및 구성 요소
xhost: 서버 접근 허용(+) 및 차단(-) 설정 명령어임 (예: xhost +는 모든 접근 허용)
xrandr: 화면 해상도를 변경할 때 사용함
디스플레이 매니저: 사용자 로그인 및 세션 관리 담당
윈도 매니저: 윈도 배치와 외형 관리 담당 (종류: Metacity, Mutter, Enlightenment, Window Maker)
데스크톱 환경: GUI 전체 환경을 제공함 (종류: GNOME, KDE, XFce)

5. 부팅 모드 및 타겟
graphical.target: X 윈도(GUI) 모드 부팅 (과거 runlevel 5)
multi-user.target: 텍스트 모드 부팅 (과거 runlevel 3). 텍스트 모드에서 X 윈도 실행 명령어는 startx임
설정 명령어: systemctl set-default로 부팅 모드를 설정하고 get-default로 확인 가능함

6. 라이브러리 및 데스크톱 환경 짝짓기
KDE: QT 라이브러리를 기반으로 함
GNOME: GTK+ 라이브러리를 기반으로 함. GNOME 2는 Metacity, GNOME 3는 Mutter를 사용함
XCB: 기존 Xlib을 대체하여 X 서버와 대화하는 저수준 라이브러리임

7. LibreOffice 제품군
Impress: 프레젠테이션 (PPT 역할)
Calc: 스프레드시트 (엑셀 역할)
Writer: 워드 프로세서 (워드 역할)

** ARP는 IP 주소를 MAC 주소로 바꿀 때 사용하며 하드웨어 주소 확인 명령으로도 출제됨
** X 윈도 원격 실행 시 '보내는 쪽'은 DISPLAY 변수 수정, '받는 쪽'은 xhost 접근 허용이 핵심임
** 부팅 모드 설정 시 set-default 뒤에 graphical.target이나 multi-user.target이 오는지 묻는 문제가 빈출됨
** GNOME(GTK+)과 KDE(QT)의 기반 라이브러리를 섞어 놓는 지문을 주의해야 함
** 리브레오피스의 각 프로그램 명칭과 역할(Impress=PPT, Calc=엑셀)을 바르게 매칭해야 함

# LAN 및 네트워크 통신 요약
1. LAN 구성 형태(Topology) 특징
버스(Bus)형: 하나의 회선에 여러 장치 연결. 모든 장치가 동일한 권한을 가지며 한 번에 한 개만 전송 가능함
스타(Star)형: 중앙 제어기를 통한 Point-to-Point 방식. 중앙 제어기 고장 시 전체 네트워크 중단됨
링(Ring)형: 전송 충돌이 없으며 노드 수가 늘어도 성능 저하가 적음. 단, 노드 추가가 어렵고 전체 중단 가능성 있음
망(Mesh)형: 장애 발생 시에도 영향이 적어 신뢰성 높음. 단, 설치 비용이 비싸고 운영 및 고장 지점 찾기가 어려움

2. 이더넷(Ethernet) 및 전송 방식
이더넷: LAN을 위해 개발된 48비트 MAC 주소 기반 통신. CSMA/CD(충돌 탐지) 방식을 사용함
회선 교환: 고정된 경로를 사용함
패킷 교환: 데이터를 패킷 단위로 동적 전송함. 오버헤드 비트가 발생하나 우선순위 부여가 가능함
FDDI: 광섬유 케이블을 사용하며 최대 100Mbps 속도를 지원하는 토큰 패싱 방식임

3. 네트워크 확인 명령어
xauth list $DISPLAY: 쿠키(COOKIE)와 인증 키값을 확인함
ip route show: 게이트웨이 주소 정보를 출력함
netstat / ss: 네트워크 연결 상태 확인 및 SYN Flooding 탐지. ss -a는 모든 소켓 정보를 출력함
ESTABLISHED: 3-Way Handshaking이 완료되어 서버와 클라이언트가 연결된 상태임

4. 웹 브라우저 및 서비스
lynx: X 윈도 없이도 사용 가능한 텍스트 기반 브라우저임
Chrome: 구글 개발. 초기 웹키트 엔진에서 현재는 블링크(Blink) 엔진을 사용함
WWW: 하이퍼텍스트 방식의 정보 검색 시스템으로 URL을 통해 웹 서버에 접근함

5. 보안 및 원격 접속 (SSH)
SSH: 패킷 암호화 기반 안전한 원격 로그인 지원. 포트 변경 시 -p 옵션 사용함
SSH 기반 서비스: scp, sftp는 SSH 기반이지만 NFS는 아님

6. 프로토콜 및 포트 번호
TCP: 3-way handshaking(SYN -> SYN/ACK -> ACK) 수행함
UDP: 수신 확인이나 오류 정정을 하지 않음 (예: DNS 서비스 등)
주요 포트: FTP(21), SSH(22), Telnet(23), SMTP(25), DNS(53), HTTP(80), POP3(110), IMAP(143), SNMP(161), HTTPS(443), SMB(445), MYSQL(3306)

** LAN 구성 형태별 장단점(중앙 고장 시 중단되는 스타형, 장애에 강한 망형 등)을 구분하는 문제가 자주 나옴
** 텍스트 기반 브라우저인 lynx와 GUI 기반 브라우저를 섞어 놓는 지문을 주의해야 함
** TCP의 연결 과정인 3-way handshaking 순서와 상태값(ESTABLISHED)의 의미가 중요함
** 주요 서비스별 포트 번호(특히 보안 포트 443, 22 등) 매칭은 필수 암기 사항임
** LAN(근거리) < MAN(대도시) < WAN(광역) 순서의 네트워크 규모 비교가 나옴

# 최신 기술 및 주요 응용 프로그램 요약
1. 자동화 및 컨테이너 기술
Ansible: 여러 운영체제에서 사용 가능한 공개 프로그램으로 2015년 레드햇이 인수함
Docker: 프로세스를 격리하는 컨테이너 기반 기술로 하이퍼바이저나 게스트 운영체제를 쓰지 않아 매우 경량화됨
Kubernetes: Docker 같은 컨테이너를 관리하는 오픈소스 플랫폼으로 CNCF에서 개발함

2. 임베디드 및 모바일 운영체제
Arduino: 하드웨어와 소프트웨어가 결합된 마이크로컨트롤러 기반의 단일 보드임
라즈베리 파이: 영국에서 교육용으로 개발된 신용카드 크기의 싱글 보드 컴퓨터임
GENIVI: 리눅스 기반의 자동차용 인포테인먼트(IVI) 플랫폼임
Tizen: 인텔과 삼성이 협력하여 만든 리눅스 커널 기반 운영체제임
webOS: Palm OS를 계승한 모바일 운영체제로 현재 LG전자에서 사용함
리눅스 커널 운영체제 종류: webOS, GENIVI, Tizen (QNX는 해당하지 않음)

3. 네트워크 및 프로토콜
IPv6: 128비트 주소 체계를 사용하며 헤더 구조가 단순하고 호스트 주소 자동 설정 및 보안 기능을 강화함 (클래스 개념 없음)
ICMP: 네트워크상에서 오류 보고 및 피드백 메시지를 전달하는 역할을 함
Ethernet: MAC 주소 기반으로 데이터를 주고받으며 시스템에서 enp0s3 같은 명칭으로 표시됨

4. 데스크톱 응용 프로그램
Evince: 리눅스용 PDF 문서 뷰어 프로그램임
파이어폭스: 모질라에서 개발했으며 게코(Gecko) 엔진을 사용함
Apache OpenOffice: oocalc(엑셀), oowriter(워드), ooimpress(PPT), oodraw(벡터 그래픽)로 구성됨
IRC: 실시간 채팅 프로토콜로 Xchat 같은 프로그램이 있음

5. 배포판 및 도메인
Rocky Linux: CentOS 지원 종료에 따라 그 대안으로 탄생한 배포판임
최상위 도메인: com, net, kr 등 도메인 체계의 가장 오른쪽에 위치하는 단위임

** Docker가 하이퍼바이저와 게스트 OS를 사용하지 않아 가볍다는 특징을 묻는 문제가 자주 나옴
** IPv6의 주소 길이(128비트)와 헤더가 단순하다는 특징을 IPv4와 비교하는 지문을 주의해야 함
** 리눅스 커널 기반의 임베디드 OS(Tizen, webOS, GENIVI)가 아닌 것(QNX 등)을 찾는 문제가 출제됨
** PDF 뷰어인 Evince와 리브레오피스/오픈오피스의 도구별 명칭(oocalc, oowriter 등) 매칭이 핵심임
** CentOS 종료 이후 등장한 배포판이 Rocky Linux라는 시사적인 내용도 기억해 두면 좋음
