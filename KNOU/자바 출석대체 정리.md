# Java 시험 벼락치기 최종 통합 정리 (컴공 시험용)

## 1. Java 특징

- 객체지향 언어(OOP)
- JVM 위에서 실행 → 운영체제 독립
- 자동 메모리 관리(Garbage Collection)
- C/C++보다 안전

실행 흐름:
.java
→ javac 컴파일
→ .class (바이트코드)
→ JVM 실행

JVM:
Java Virtual Machine

---

### 변수 선언 / 생성 / 초기화

선언만:
int x;

선언 + 초기화:
int x = 10;

선언 후 초기화:
int x;
x = 10;

객체 선언:
Member m;

객체 생성:
m = new Member();

선언 + 생성:
Member m = new Member();

지역변수:
초기화 안 하면 사용 불가

멤버변수(필드):
자동 초기화

int → 0
double → 0.0
boolean → false
String → null

## 5. 클래스(Class)

구성:
- 필드(Field)
- 생성자(Constructor)
- 메서드(Method)

---

### 생성자(Constructor)

객체 생성 시 자동 호출

특징:
클래스명과 동일
반환타입 없음

예:
Person(){}

---

### this

현재 객체 참조

this.name = name;

---

### static

객체 생성 없이 사용

공유 자원

예:
Math.random()

static 변수:
공유 변수

static 메서드:
객체 없이 호출 가능

---

### final

수정 불가

final 변수:
상수

final 메서드:
오버라이딩 금지

final 클래스:
상속 금지

---

## 6. 상속(Inheritance)

---

### super

부모 객체 접근

super();

---

### 오버로딩 (Overloading)

메서드 이름 같음
매개변수 다름

예:
add(int a)
add(int a, int b)

---

### 오버라이딩 (Overriding)

상속받은 메서드 재정의

조건:
메서드 이름 동일
매개변수 동일

---

## 7. 접근 제어자 ⭐

public
→ 어디서나 접근 가능

protected
→ 같은 패키지 + 상속

default
→ 같은 패키지

private
→ 같은 클래스만

접근 범위:

public
> protected
> default
> private

인터페이스 → implements

오버로딩:
이름 같고 매개변수 다름

오버라이딩:
재정의

다형성:
부모 = 자식 객체 가능

== → 주소 비교
equals → 내용 비교

List:
순서 O 중복 O

Set:
순서 X 중복 X

Map:
key-value

try-catch-finally
throw / throws 차이

Generic → 타입 안정성
Lambda → 익명 함수

static → 객체 없이 사용
final → 수정 불가
this → 현재 객체
super → 부모 객체
