컴퓨터 보안
1강 컴퓨터 보안의 개념
컴퓨터 보안 : 컴퓨팅 환경과 관련된 모든 정보를 보호하는 것
※ 정보보호 목표(CIA)
- 기밀성(Confidentiality) : 허가되지 않은 사람이 정보를 알 수 없도록 보호
- 무결성(Integrity) : 허가 없이 정보가 변경되지 않도록 보호
- 가용성(Availability) : 허가된 사용자가 필요할 때 사용할 수 있도록 보장
추가 목표 : 부인방지 / 인증(Authentication) / 접근제어(Access Control)
정보화 환경과 역기능
- 정보 전파 속도 증가
- 새로운 공격 기법 지속 등장
- 피싱 → 보이스피싱 → 스미싱 등으로 발전
※ 컴퓨터 보안 역사
1950년대 해커 → ARPANET → TCP/IP → DDoS

2강 암호의 정의
암호 : 안전하지 않은 채널에서도 제3자가 내용을 알 수 없도록 보호하는 기술
키(Key) : 암호화와 복호화에 사용하는 열쇠
고대 암호
- 스테가노그래피 : 정보를 숨겨 전달
전치법
- 스파르타 봉 암호 : 평문의 문자 순서를 변경
치환법
- 시저 암호 : 알파벳을 3칸 이동
- 시프트 암호 : 알파벳을 K칸 이동
- 비즈네르 암호 : 여러 개의 키를 사용하는 시프트 암호
근대 암호
※ 섀넌
- 혼돈(Confusion) : 평문과 암호문의 관계를 숨김
- 확산(Diffusion) : 평문의 통계적 특성을 암호문 전체에 분산
현대 암호 : DES(1977) / AES(2001)
대칭키 암호
- 같은 키로 암호화와 복호화 수행
- 장점 : 속도가 빠름
- 단점 : 키 분배 문제
암호 방식 : 블록 암호(블록 단위) / 스트림 암호(XOR 이용)
대칭키 알고리즘 : DES / AES / IDEA
공개키 암호
- 서로 다른 키 사용
- 공개키로 암호화, 개인키로 복호화
- 장점 : 키 관리와 분배가 쉬움
- 단점 : 속도가 느림
공개키 알고리즘
- RSA : 소인수분해 문제 기반
- ECC : 타원곡선 이산대수 문제 기반
- ElGamal : 이산대수 문제 기반

3강 인증의 개념
인증 : 실체가 실제 그 대상이 맞는지 확인하는 과정
메시지 인증
- 메시지가 전송 중 변경되지 않았는지 확인
- 무결성을 보장
MAC(Message Authentication Code)
- 메시지 위변조 여부를 확인하는 코드
- 비밀키를 이용하여 생성
MAC 종류 : HMAC(해시 함수 기반) / CMAC(블록 암호 기반)
사용자 인증
- 시스템 접근 사용자의 신원 확인
사용자 인증 방식 : 비밀번호 / 생체인식 / 토큰 / 2단계 인증

4강 사이버 공격의 개요
사이버 공격 : 인터넷을 통해 시스템에 피해를 입히는 행위
악성코드
- 목적 : 시스템 손상, 전파, 정보 유출
악성코드 종류
- 바이러스 : 다른 파일에 기생하여 전파
- 웜 : 스스로 복제하여 네트워크로 전파
- 트로이목마 : 정상 프로그램으로 위장
- 백도어 : 비정상적인 우회 접근 통로
- 스파이웨어 : 사용자 정보 수집
- 랜섬웨어 : 데이터를 암호화하고 금전 요구
네트워크 공격
- 스캐닝 : 공격 전 정보 수집
- 스푸핑 : 신뢰하는 대상으로 위장
- 스니핑 : 네트워크 데이터 도청
- DoS : 대량 요청으로 서비스 마비
- SYN Flooding : TCP 연결을 과도하게 생성
- DDoS : 여러 공격자가 동시에 서비스 거부 공격
- 피싱 : 개인정보나 금융정보 탈취
- 피밍 : 도메인을 조작하여 가짜 사이트로 유도
최근 사이버 공격
- 에이전트화 : 공격 프로그램 활용
- 분산화 : 여러 시스템을 이용한 공격
- 자동화 : 공격 과정 자동 수행
- 은닉화 : 공격 흔적 숨김
- APT : 특정 대상을 장기간 은밀하게 공격

5강 서버 보안의 개요
서버 보안 계층
- 응용 프로그램 계층 : 백도어, 버그 등을 이용해 운영체제 접근 권한 획득
- 운영체제 계층 : 시스템 제어권 확보
- 네트워크 제어 : 네트워크 트래픽 관리
- 전송 데이터 제어 : 데이터를 암호화하여 전송
서버 공격 유형
- 계정 크랙 공격 : 계정과 비밀번호 탈취
- 네트워크 공격 : 네트워크 취약점 이용
- 버퍼 오버플로 : 메모리 영역을 넘쳐 악성코드 실행
- 사회공학적 공격 : 사람을 속여 정보 탈취
서버 보안 대책
- 계정 관리 : 그룹별 접근권한 부여, 불필요한 계정 삭제
- 패스워드 관리 : 유추하기 어려운 문자 조합 사용
- 파일 시스템 보호 : 접근 권한 관리, 암호화, 백업 및 복구
- 시스템 파일 관리 : 중요 파일 보호 및 설정
- 시스템 로그 관리 : 로그 설정과 분석
시스템 접근제어
- DAC(임의적 접근제어) : 소유자가 권한 설정
- MAC(강제적 접근제어) : 보안 등급에 따라 권한 설정
- RBAC(역할기반 접근제어) : 역할에 따라 권한 부여
운영체제 관리
- 작업 용도별 파티션 분리
- 불필요한 기능 제거
- 최소 권한으로 프로세스 수행
시스템 로그 : UNIX(syslog) / Windows(이벤트 뷰어)
서버 관리자 역할
- 시스템 시작과 종료
- 사용자 계정 관리
- 프로세스·메모리·디스크 관리
- 네트워크 연결 및 상태 관리

6강 네트워크 보안의 개요
컴퓨터 네트워크 : 상호 접속을 통해 정보를 교환하는 환경
목적 : 통신 회선상의 정보를 안전하게 보호
네트워크 위협
- 물리적 공격 : 장비 파괴, 손상, 도난
- 기술적 공격 : 수동적 공격과 능동적 공격
※ 네트워크 보안 목표
기밀성 / 무결성 / 가용성 / 부인방지 / 사용자 인증 / 데이터 발신처 확인 / 접근제어
네트워크 보안 모델
보안 서비스
- 인증 : 사용자나 시스템의 신원 확인
- 접근제어 : 허가된 접근만 허용
- 기밀성 : 정보 노출 방지
- 데이터 무결성 : 데이터 변조 방지
- 부인방지 : 행위 부인 방지
보안 메커니즘 : 암호화 / 전자서명 / 접근제어 / 데이터 무결성 / 인증교환 / 트래픽 패딩 / 라우팅 제어 / 공증
LAN 보안
- 개방형 구조로 정보 유출, 데이터 변경, 순서 변경, 바이러스 감염 등에 취약
- 제공 서비스 : 데이터 발신처 인증 / 기밀성 / 비접속 무결성 / 접근제어
TCP/IP 보안
IPsec
- IP 통신을 위한 보안 기술
IPsec 프로토콜
- AH(Authentication Header) : 송신자 인증 제공
- ESP(Encapsulating Security Payload) : 송신자 인증 + 데이터 암호화
SSL/TLS
- SSL : 서버·클라이언트 인증과 기밀성 제공
- TLS : SSL 3.0을 개선한 프로토콜

7강·8강 보안 시스템의 개요
보안 시스템 종류
Firewall / IDS / IPS / VPN / NAC / UTM
Firewall(방화벽)
- 내부 네트워크를 외부 공격으로부터 보호
IDS(침입탐지시스템)
- 기밀성·무결성·가용성을 해치는 행위를 실시간 탐지
- 공격은 탐지만 하고 차단하지 않음
IDS 구성
- 모니터링부 : 정보와 데이터 수집
- 분석 및 조치부 : 침입 여부 분석
- 관리부 : 정책 관리와 시스템 통제
IDS 종류
- 응용기반 : 애플리케이션 정보 수집
- 호스트기반 : 시스템 행위 감시
- 목표기반 : 데이터와 프로세스 무결성 분석
- 네트워크기반 : 패킷을 수집하여 탐지
- 통합방식 : 여러 탐지 방식을 결합
IPS(침입방지시스템)
- 공격을 실시간 탐지하고 자동 대응
- 호스트 기반 / 네트워크 기반
※ IDS : 탐지만 수행
※ IPS : 탐지 + 차단
VPN(가상사설망)
- 공중망을 사설망처럼 사용하는 기술
- 목적 : 저렴한 비용으로 보안성과 성능 향상
VPN 기능 : 보안 및 정보보호 / 서비스 품질(QoS) / 신뢰성 / 보안관리 / 다중 서비스 지원

9강 메일 보안의 개요
메일
- 여러 호스트를 거쳐 전달되므로 내용 노출 가능
PGP(Pretty Good Privacy)
- 전자우편 보안 도구
PGP 기능
- 인증 : SHA + RSA 전자서명
- 기밀성 : IDEA 암호화 + RSA로 세션키 전달
- 압축 : 전송 전 압축 수행
PGP 전송 과정
- 송신자 : 서명 생성 → 압축 → 암호화 → 세션키 첨부
- 수신자 : 세션키 복구 → 복호화 → 압축 해제 → 해시 비교
S/MIME
- MIME 기반 전자우편 보안 서비스
MIME
- 다양한 형태의 데이터를 전자우편으로 전송하기 위한 형식
S/MIME 기능
- 전자서명
- 서명된 영수증
- 보안 레이블
- 보안 메일링 리스트

10강 웹 보안의 개요
웹 보안
- 웹 서버와 클라이언트 간 서비스를 보호하는 기술
- SSL/TLS를 이용하여 보안 제공
웹 공격
- SQL Injection : SQL문을 삽입하여 인증 우회 및 데이터 탈취
XSS(Cross Site Scripting)
- 악성 스크립트를 실행하는 공격
XSS 종류
- 저장형 XSS : 게시판 등에 저장 후 실행
- 반사형 XSS : URL 등을 통해 즉시 실행
접근제어 실패
- 권한 검사가 제대로 이루어지지 않는 취약점
- 관리자 페이지 노출
- URL에 사용자 정보 포함
웹 서버 공격
- 디렉터리 목록 노출 : 서버 내부 파일 목록이 외부에 공개되는 취약점
모바일 보안
무선 네트워크
- LAN : Wi-Fi
- WAN : 이동통신망
모바일 보안 기법
- WEP : RC4 알고리즘 사용
- RSN : 인증, 키관리, 기밀성, 무결성 제공
- TKIP : 임시 키 무결성 프로토콜
- CCMP : AES 기반 기밀성과 무결성 제공
- EAP : 확장형 인증 프레임워크
WPA
- Enterprise : 인증 서버 사용
- Personal : PSK(사전 공유키) 사용

11강 디지털 포렌식 개요
디지털 포렌식 : 범죄 사실 규명을 위해 디지털 증거를 수집·분석하는 기술
포렌식 수행 과정
- 디지털 데이터 수집
- 데이터 분석 및 증거 확보
- 조사 결과 보고서 작성
디지털 증거
증거 유형
- 내용물 정보 : 사건을 직접 증명하는 정보(컴퓨터 저장 증거)
- 특성정보 : 증거를 식별·분류하는 메타데이터(컴퓨터 생성 증거)
증거 특성
- 비가시성 : 육안으로 확인 불가
- 변조 가능성 : 훼손·조작이 쉬움
- 복제 용이성 : 원본과 동일하게 복제 가능
- 대규모성 : 필요한 증거를 찾기 어려움
- 휘발성 : 메모리·네트워크에서 일시적으로 존재
- 초국경성 : 인터넷을 통해 국가를 넘나듦
법적 허용성 요건
- 인증성 : 정확하게 수집되었는가
- 무결성 : 변경 없이 보존되었는가
- 신뢰성 : 장비·프로그램·분석 과정이 신뢰되는가
- 원본성 : 원 데이터와 동일한가
안티 포렌식
- 증거를 은닉·삭제·변조하여 포렌식 분석을 방해하는 기술

12강 대칭키 암호의 개념
대칭키 암호 : 하나의 같은 비밀키로 암호화와 복호화를 수행
블록 암호
- 평문을 고정 크기 블록으로 나누어 암호화
블록 암호 구성 요소
- 라운드 함수 : 반복 수행되는 함수
- 라운드 키 : 각 라운드에서 사용하는 키
- 키 스케줄 : 라운드 키 생성 과정
블록 암호 구조
- 파이스텔 구조 : 입력을 좌우로 나누어 암호화, 역변환이 쉬움
- SPN 구조 : 여러 소블록으로 나누어 암호화, 역함수 필요
블록 암호 사용 모드
- ECB : 블록별 독립 암호화, 오류 전파 없음
- CBC : 이전 암호문 블록이 다음 블록에 영향
- CFB : 키 스트림 방식으로 동작, 스트림 암호처럼 사용
- OFB : 오류가 한 블록에만 영향
- CTR : 병렬 처리 가능, 오류 전파 없음
스트림 암호
- 키 스트림과 평문을 XOR하여 암호화
LFSR
- 선형 귀환 시프트 레지스터
- 쉽게 해독될 수 있어 비선형 결합 사용
대칭키 알고리즘 : DES / 3DES(TDEA) / AES
AES
- SPN 구조 기반

- ElGamal : 이산대수 기반

데이터베이스
2강 데이터베이스 모델링
주요 용어
- 사용자 요구사항 분석 : 사용자가 원하는 데이터와 기능을 파악하는 과정
- 데이터 모델링 : 데이터 구조와 관계를 정의하여 데이터베이스를 설계하는 과정
- 개념적 데이터 모델링 : 실세계를 추상화하여 개체, 속성, 관계를 도출하는 과정
- 논리적 데이터 모델링 : 데이터를 논리적 구조로 설계하는 과정
- 물리적 데이터 모델링 : 실제 DBMS에 맞게 저장 구조를 설계하는 과정
- ER 모델 : 개체(Entity)와 관계(Relationship)를 이용하여 데이터를 표현하는 모델
- 개체 집합 : 공통된 속성을 가진 개체들의 집합
- 관계 집합 : 개체 집합 간의 연결 관계

핵심 정리
- 데이터베이스 모델링 순서 : 사용자 요구사항 분석 → 개념적 데이터 모델링 → 논리적 데이터 모델링 → 물리적 데이터 모델링
- 요구사항이 모호하거나 정확하지 않으면 이전 단계로 돌아가 수정한다.
- ER 모델은 개념적 데이터 모델링의 대표적인 도구이며 전체 스키마 설계에 사용된다.
- ER 모델의 기본 요소는 개체(Entity), 속성(Attribute), 관계(Relationship)이다.
- 속성의 종류 : 단순 속성, 복합 속성, 단일값 속성, 다중값 속성, 저장 속성, 유도 속성, 널 속성
- ER 모델의 제약조건
  - 사상수 : 1:1, 1:N, N:1, M:N
  - 참가 제약조건 : 전체적 참여, 부분적 참여
  - 키 속성 : 개체를 유일하게 식별하는 최소 속성 집합
- 재귀적 관계 집합 : 동일한 개체 집합이 서로 다른 역할로 관계를 맺는 것
- 약한 개체 집합 : 독립적으로 존재할 수 없으며 강한 개체 집합과 1:N 관계를 가져야 하고, 키는 강한 개체의 키와 부분키로 구성된다.

시험 암기
- 모델링 순서 : 요구사항 분석 → 개념 → 논리 → 물리
- ER 모델 구성 요소 : 개체 / 속성 / 관계
- 사상수 : 1:1 / 1:N / N:1 / M:N
- 참가 제약조건 : 전체 참여 / 부분 참여
- 약한 개체 키 : 지배 개체 키 + 부분키
- 재귀적 관계 : 동일한 개체 집합이 서로 다른 역할 수행

3강 관계형 모델
주요 용어
- 릴레이션 : 관계형 모델에서 데이터를 저장하는 2차원 표 형태의 구조
- 릴레이션 스키마 : 테이블의 컬럼과 데이터 타입을 정의한 구조
- 키(Key) : 레코드를 유일하게 식별하는 값으로 유일성과 최소성을 만족
- 제약조건 : 데이터의 무결성과 일관성을 유지하기 위해 반드시 만족해야 하는 조건
- 관계 대수 : 필요한 데이터를 연산자로 표현하는 절차적 질의 언어

핵심 정리
- 관계형 모델은 릴레이션(테이블)을 기반으로 데이터를 저장한다.
- 레코드(Tuple)는 행(Row), 컬럼(Attribute)은 열(Column)을 의미한다.
- 릴레이션의 특징
  - 레코드의 유일성
  - 레코드의 무순서성
  - 컬럼의 무순서성
  - 컬럼값의 원자성
- 기본키(PK)는 레코드를 유일하게 식별하는 컬럼이다.
- 카디널리티(Cardinality)는 레코드의 개수이다.
- 키의 종류 : 기본키(PK), 후보키(Candidate Key), 수퍼키(Super Key)
- 관계형 모델은 데이터의 삽입, 수정, 삭제 시 제약조건을 검사하여 무결성을 유지한다.
- 제약조건의 종류
  - 영역 제약조건
  - 키 제약조건
  - 개체 무결성 제약조건
  - 참조 무결성 제약조건
- ER 모델을 실제 DBMS 구조로 변환하는 과정을 논리적 데이터 모델링이라고 한다.
- ER 모델 변환
  - 개체 → 릴레이션
  - 약한 개체 → 강한 개체의 키 + 부분키를 기본키로 하는 릴레이션
  - 사상수는 기본키와 외래키의 참조 관계로 표현
- 관계 데이터베이스는 관계 대수를 이용하여 데이터를 검색한다.
- 관계 대수 연산
  - 셀렉트(Select)
  - 프로젝트(Project)
  - 합집합(Union)
  - 교집합(Intersection)
  - 차집합(Difference)
  - 카티션 프로덕트(Cartesian Product)
  - 자연 조인(Natural Join)
  - 할당 연산
  - 집계 함수(SUM, AVG, MAX, MIN, COUNT)

시험 암기
- 릴레이션 = 2차원 표
- 레코드 = 행(Row), 컬럼 = 열(Column)
- 릴레이션 특징 : 유일성 / 레코드 무순서 / 컬럼 무순서 / 원자성
- 키 조건 : 유일성 + 최소성
- 카디널리티 = 레코드 개수
- ER 모델 변환 : 개체→릴레이션, 관계→PK/FK
- 제약조건 : 영역 / 키 / 개체 무결성 / 참조 무결성
- 관계 대수 주요 연산 : Select, Project, Join, Union, Difference, Cartesian Product

8강 정규화
주요 용어
- 정규화 : 이상현상을 방지하기 위해 원칙에 따라 스키마를 분해하는 기법
- 함수적 종속성(Functional Dependency) : 한 속성의 값이 다른 속성의 값을 결정하는 관계
- 클로저(Closure) : 추론규칙을 이용해 도출한 모든 함수적 종속성의 집합
- 카노니컬 커버(Canonical Cover) : 자명성과 중복성을 제거한 최소 함수적 종속성 집합
- 정규형(Normal Form) : 이상현상을 방지하기 위해 릴레이션을 일정한 조건에 따라 재구성한 형식

핵심 정리
- 데이터베이스 설계 시 어떤 속성들을 하나의 테이블로 구성할지 결정하는 것이 중요하다.
- 정규화의 목적
  - 데이터 중복 제거
  - 저장 공간 낭비 최소화
  - 데이터 추가 시 수정 최소화
  - 삽입, 삭제, 수정 이상(Update Anomaly) 방지
- 이상현상의 종류
  - 삽입 이상
  - 삭제 이상
  - 수정 이상
- 함수적 종속성은 릴레이션 속성 간의 기본적인 제약조건이며 정규화의 기준이 된다.
- 릴레이션 인스턴스만으로 모든 함수적 종속성을 찾기 어려우므로 암스트롱 공리를 이용하여 클로저를 구한다.
- 클로저에는 불필요한 종속성이 포함될 수 있으므로 자명성과 중복성을 제거한 카노니컬 커버를 이용하여 정규화를 수행한다.
- 정규형은 함수적 종속성과 추론 규칙을 바탕으로 구성된다.
- 정규화는 단계가 올라갈수록 제약조건이 강화된다.
- 정규형 순서
  - 제1정규형(1NF) - 제2정규형(2NF) - 제3정규형(3NF) - BC정규형(BCNF) 
  - 제4정규형(4NF) - 제5정규형(5NF)

시험 암기
- 정규화 목적 : 중복 제거 / 이상현상 방지 / 저장공간 절약
- 이상현상 : 삽입 이상 / 삭제 이상 / 수정 이상
- 함수적 종속성 : X → Y (X가 Y를 결정)
- 클로저 : 암스트롱 공리로 확장한 함수적 종속성 집합
- 카노니컬 커버 : 최소화된 함수적 종속성 집합
- 정규형 순서 : 1NF → 2NF → 3NF → BCNF → 4NF → 5NF
- 정규화가 진행될수록 제약조건은 강화된다.

9강 저장 객체
주요 용어
- 저장 객체 : 데이터베이스 내에 저장되어 프로그램과 같은 기능을 수행하는 SQL 확장 기능
- 저장 프로시저(Procedure) : 자주 사용하거나 복잡한 SQL문을 저장하여 사용하는 저장 객체
- 함수(Function) : 특정 값을 반환하며 SELECT문에서 사용할 수 있는 저장 객체
- 트리거(Trigger) : INSERT, UPDATE, DELETE 이벤트 발생 시 자동 실행되는 저장 객체
- 매개변수(Parameter) : 저장 객체에 값을 전달하거나 반환받기 위한 변수(IN, OUT, INOUT)

핵심 정리
- 저장 객체는 비절차적 SQL의 한계를 보완하여 절차적 처리가 가능하도록 만든 데이터베이스 프로그래밍 요소이다.
- 저장 객체의 종류 : 저장 프로시저, 함수, 트리거
- 저장 객체의 장점
  - 네트워크 전송 효율 향상
  - 실행 속도 향상
  - 모듈화
  - 보안 강화
- 저장 객체의 단점
  - 처리 성능 저하 가능
  - 디버깅이 어려움
- MySQL 변수의 종류
  - 사용자 정의 변수 : @변수명 (세션 전체 사용)
  - 로컬 변수 : DECLARE (저장 객체 내부에서만 사용)
- 저장 프로시저
  - CREATE PROCEDURE로 생성
  - CALL 명령으로 실행
  - 매개변수 : IN, OUT, INOUT
- 저장 객체에서는 조건문과 반복문을 사용할 수 있다.
  - 조건문 : IF-THEN, CASE WHEN
  - 반복문 : WHILE, REPEAT, LOOP
- 함수는 RETURN 문으로 값을 반환하며 SELECT문에서 사용할 수 있다.
- 트리거는 INSERT, UPDATE, DELETE 발생 시 자동 실행되어 데이터 무결성과 일관성을 유지한다.
- 트리거 종류
  - BEFORE INSERT
  - AFTER INSERT
  - BEFORE UPDATE
  - AFTER UPDATE
  - BEFORE DELETE
  - AFTER DELETE
- 트리거에서는 OLD와 NEW를 이용하여 변경 전후 데이터를 참조한다.
- 저장 객체 수정은 일반적으로 DROP 후 CREATE를 사용하며, ALTER는 일부 속성 변경에 사용된다.
- MySQL Workbench에서는 GUI 방식으로 저장 객체를 관리할 수 있다.

시험 암기
- 저장 객체 종류 : 프로시저 / 함수 / 트리거
- 프로시저 : CALL 실행, 값 반환 필수 아님
- 함수 : RETURN으로 값 반환, SELECT문 사용 가능
- 트리거 : INSERT / UPDATE / DELETE 시 자동 실행
- 매개변수 : IN / OUT / INOUT
- 변수 : @변수(세션) / DECLARE(로컬)
- 트리거 데이터 참조 : OLD / NEW
- 저장 객체 수정 : DROP 후 CREATE

10강 데이터 저장과 파일
주요 용어
- 파일(File) : 디스크에 데이터를 저장하고 관리하는 가장 기본적인 논리적 저장 단위
- 블록(Block) : 디스크에 데이터를 저장하는 고정 크기의 물리적 저장 단위
- 레코드(Record) : 블록을 구성하는 최소 데이터 저장 단위
- 멀티셋(Multiset) : 하나의 레코드에서 하나의 컬럼이 여러 값을 가지는 형태

핵심 정리
- 저장장치는 데이터 접근 속도에 따라 계층적으로 구성된다.
- 상위 계층 저장장치
  - 접근 속도 빠름
  - 가격 비쌈
  - 대부분 휘발성
- 하위 계층 저장장치
  - 접근 속도 느림
  - 가격 저렴
  - 대부분 비휘발성
- 데이터베이스는 여러 개의 파일로 구성되며, 파일은 여러 개의 블록으로 저장된다.
- 하나의 파일은 여러 개의 레코드로 구성된다.
- 레코드 형식
  - 고정 길이 레코드
  - 가변 길이 레코드
- 슬롯 페이지 구조는 가변 길이 레코드를 효율적으로 관리하기 위한 블록 구조이다.
- 파일 구조화 방식
  - 힙 파일 구조화 : 저장 순서와 관계없이 저장
  - 순차 파일 구조화 : 특정 순서대로 저장
  - 해시 파일 구조화 : 해시 함수를 이용해 저장
  - 다중 테이블 군집 파일 구조 : 여러 테이블을 하나의 파일에서 관리
- 메모리와 디스크는 블록 단위로 데이터를 주고받는다.
- 데이터베이스는 블록 입출력을 줄이기 위해 메모리 내에 버퍼(Buffer)를 사용한다.
- 버퍼 관리자는 버퍼 공간을 관리하며, 공간이 부족하면 버퍼 교체 전략을 사용하여 블록을 디스크로 내보낸다.

시험 암기
- 파일 : 논리적 저장 단위
- 블록 : 물리적 저장 단위
- 레코드 : 최소 데이터 저장 단위
- 저장장치 특징 : 상위(고속·고가·휘발성) / 하위(저속·저가·비휘발성)
- 레코드 종류 : 고정 길이 / 가변 길이
- 가변 길이 레코드 관리 : 슬롯 페이지 구조
- 파일 구조화 : 힙 / 순차 / 해시 / 다중 테이블 군집
- 메모리 ↔ 디스크 : 블록 단위 전송
- 블록 입출력 최소화 : 버퍼 사용
- 버퍼 공간 관리 : 버퍼 관리자, 버퍼 교체 전략

11강 인덱싱
주요 용어
- 탐색키(Search Key) : 파일에서 원하는 레코드를 찾기 위해 사용하는 컬럼 또는 컬럼값의 집합
- 인덱스 엔트리(Index Entry) : 탐색키와 해당 레코드의 포인터를 함께 저장한 구조
- 다단계 인덱스(Multi-Level Index) : 인덱스를 여러 단계로 구성하여 인덱스 파일 크기를 줄인 구조
- 이진 탐색 트리(Binary Search Tree) : 왼쪽 자식은 작은 값, 오른쪽 자식은 큰 값을 가지는 탐색 트리

핵심 정리
- 데이터베이스는 데이터 검색 속도를 높이기 위해 인덱싱과 해싱을 사용한다.
- 인덱스는 데이터 파일의 빠른 탐색을 위한 부가적인 자료구조이며, 이를 생성하는 과정을 인덱싱이라고 한다.
- 순차 파일에서는 탐색키 순으로 정렬되어 있어 순서 인덱스를 이용한 빠른 임의 접근이 가능하다.
- 인덱스 종류
  - 밀집(Dense) 인덱스 : 모든 레코드에 대해 인덱스 엔트리 생성
  - 희소(Sparse) 인덱스 : 일부 레코드에 대해서만 인덱스 엔트리 생성
  - 다단계(Multi-Level) 인덱스 : 밀집 인덱스와 희소 인덱스를 결합한 구조
- B+트리는 대표적인 인덱스 구조로 높이 균형 트리(Balanced Tree)이다.
- B+트리는 루트에서 리프(Leaf) 노드까지 모든 경로의 길이가 같다.
- B+트리 탐색은 루트 노드부터 시작하여 비교 연산을 거쳐 리프 노드에서 종료된다.
- 레코드 삽입과 삭제 시 B+트리는 균형을 유지하기 위해 재구조화된다.
- B+트리에서는 인덱스 엔트리 삽입과 삭제 시 노드 분할(Split)과 노드 병합(Merge)이 발생할 수 있다.

시험 암기
- 인덱스 목적 : 데이터 검색 속도 향상
- 탐색 방법 : 인덱싱 + 해싱
- 인덱스 엔트리 = 탐색키 + 레코드 포인터
- 인덱스 종류 : 밀집 / 희소 / 다단계
- 밀집 인덱스 : 모든 레코드
- 희소 인덱스 : 일부 레코드
- 다단계 인덱스 : 밀집 + 희소
- B+트리 특징
  - 높이 균형 트리
  - 루트~리프 경로 길이 동일
  - 탐색은 리프 노드에서 종료
- B+트리 갱신 : 삽입·삭제 시 노드 분할과 병합 발생

12강 해싱과 특수 인덱스
주요 용어
- 해시 함수(Hash Function) : 탐색키를 버킷 주소에 대응시키는 함수
- 버킷(Bucket) : 하나 이상의 레코드를 저장할 수 있는 저장 공간
- 충돌(Collision) : 서로 다른 탐색키가 같은 버킷에 저장되는 현상
- 오버플로(Overflow) : 버킷의 저장 공간이 부족한 상태
- 비트맵(Bitmap) : 레코드 존재 여부를 비트(0,1)로 표현한 비트열

핵심 정리
- 해싱은 수학적 함수를 이용하여 데이터를 빠르게 검색하는 기법이다.
- 해싱의 종류
  - 정적 해싱 : 버킷 개수가 고정
  - 동적 해싱 : 데이터 양에 따라 버킷 개수가 변함
- 해시 함수는 탐색키와 버킷 주소를 연결하는 역할을 하며, 레코드가 균등하게 분산되는 것이 가장 이상적이다.
- 서로 다른 탐색키가 같은 버킷에 저장되는 현상을 충돌이라고 하며, 같은 버킷에 저장된 레코드를 동거자(Synonym)라고 한다.
- 충돌이 많이 발생하여 버킷에 더 이상 저장할 수 없는 상태를 오버플로라고 한다.
- 확장성 해싱은 대표적인 동적 해싱 기법으로 데이터 증가에 따라 버킷을 확장한다.
- 확장성 해싱은 디렉토리와 버킷으로 구성되며, 모조키(Pseudo Key)를 이용해 버킷을 관리한다.
- 비트맵 인덱스는 다중 조건 검색을 효율적으로 처리하기 위한 특수 인덱스이다.
- 비트맵은 레코드 수만큼의 비트 배열로 구성된다.
- 비트맵 인덱스는 여러 조건의 비트열에 대해 AND 연산을 수행하여 원하는 레코드를 빠르게 찾는다.

시험 암기
- 해싱 목적 : 데이터 검색 속도 향상
- 해싱 종류 : 정적 해싱 / 동적 해싱
- 해시 함수 : 탐색키 → 버킷 주소
- 충돌 : 서로 다른 탐색키가 같은 버킷 사용
- 동거자 : 같은 버킷에 저장된 레코드
- 오버플로 : 버킷 저장 공간 부족
- 확장성 해싱 : 디렉토리 + 버킷 + 모조키
- 비트맵 인덱스 : 다중 조건 검색에 사용
- 비트맵 검색 : 비트 AND 연산 수행

13강 트랜잭션
주요 용어
- 트랜잭션(Transaction) : 데이터베이스를 처리하는 하나의 논리적 작업 단위
- ACID 특성 : 트랜잭션의 안전성을 보장하는 특성(원자성, 일관성, 고립성, 지속성)
- 동시성 제어(Concurrency Control) : 다중 사용자 환경에서 데이터의 일관성을 유지하도록 트랜잭션을 제어하는 기능
- 직렬 스케줄(Serial Schedule) : 하나의 트랜잭션이 모두 끝난 후 다음 트랜잭션이 실행되는 스케줄
- 병렬 스케줄(Parallel Schedule) : 하나의 트랜잭션이 끝나기 전에 다른 트랜잭션이 함께 실행되는 스케줄

핵심 정리
- 트랜잭션은 중간에 분할할 수 없는 최소 작업 단위이다.
- DBMS는 ACID 특성을 만족하도록 트랜잭션과 데이터 상태를 관리한다.
- ACID 특성
  - 원자성(Atomicity) : 모두 수행하거나 모두 취소
  - 일관성(Consistency) : 실행 전후 데이터의 일관성 유지
  - 고립성(Isolation) : 동시에 실행되어도 서로 영향을 주지 않음
  - 지속성(Durability) : 커밋된 결과는 영구적으로 저장
- 트랜잭션 상태
  - 동작(Active)
  - 부분 커밋(Partially Committed)
  - 실패(Failed)
  - 중단(Aborted)
  - 커밋(Committed)
- 다중 사용자 환경에서는 동일한 데이터에 동시에 접근할 수 있으므로 동시성 제어가 필요하다.
- 동시성 제어의 목적은 데이터 무결성과 트랜잭션의 일관성을 유지하는 것이다.
- 스케줄이 직렬 스케줄과 동일한 결과를 낼 수 있으면 직렬 가능(Serializable)하다고 한다.
- 트랜잭션이 실패하면 시작 전 상태로 되돌리는 과정을 회복(Recovery)이라고 한다.
- 다른 트랜잭션이 수정한 데이터를 읽는 경우, 데이터를 수정한 트랜잭션이 먼저 커밋되어야 하며 이를 회복 가능 스케줄(Recoverable Schedule)이라고 한다.

시험 암기
- 트랜잭션 : 최소 논리적 작업 단위
- ACID : 원자성 / 일관성 / 고립성 / 지속성
- 트랜잭션 상태 : 동작 → 부분 커밋 → 커밋
                           ↘ 실패 → 중단
- 동시성 제어 목적 : 무결성 유지 + 일관성 유지
- 직렬 스케줄 : 순차 실행
- 병렬 스케줄 : 동시 실행
- 직렬 가능성 : 직렬 스케줄과 같은 결과
- 회복 : 실패 시 이전 상태로 복구
- 회복 가능 스케줄 : 수정한 트랜잭션이 먼저 커밋

14강 동시성 제어
주요 용어
- 양립성 함수(Compatibility Function) : 락의 종류에 따라 동시에 허용 가능한지 판단하는 함수
- 2단계 락킹 규약(2PL) : 락 획득 단계와 락 반납 단계로 구분하여 직렬성을 보장하는 규약
- 토마스 기록 규칙(Thomas Write Rule) : 타임스탬프 기반에서 오래된 쓰기 연산을 무시하는 규칙
- 교착상태(Deadlock) : 여러 트랜잭션이 서로의 작업이 끝나기를 기다리며 무한 대기하는 상태

핵심 정리
- 동시성 제어는 여러 트랜잭션이 동시에 동일한 데이터에 접근할 때 데이터의 무결성과 일관성을 유지하기 위한 기법이다.
- 동시성 제어의 대표적인 방법
  - 락 기반 규약(Lock-Based Protocol)
  - 타임스탬프 순서 규약(Timestamp Ordering)
  - 검증 기반 규약(Validation Protocol)
- 락은 한 트랜잭션이 데이터에 접근하는 동안 다른 트랜잭션의 접근을 제어하는 기법이다.
- 락의 종류
  - 공유 락(Shared Lock, S-Lock) : 읽기 가능, 서로 양립 가능
  - 배타 락(Exclusive Lock, X-Lock) : 읽기·쓰기 가능, 다른 어떤 락과도 양립 불가
- 양립성
  - 공유 ↔ 공유 : 가능
  - 공유 ↔ 배타 : 불가능
  - 배타 ↔ 배타 : 불가능
- 2단계 락킹 규약
  - 요청 단계(Growing Phase) : 락 획득만 가능
  - 반납 단계(Shrinking Phase) : 락 해제만 가능, 새로운 락 요청 불가
- 타임스탬프 순서 기법은 각 트랜잭션에 고유한 타임스탬프를 부여하여 직렬 가능성을 보장한다.
- TS(T1) < TS(T2)이면 T1이 먼저 수행된 것처럼 처리한다.
- 토마스 기록 규칙은 오래된 쓰기 연산을 무시하여 불필요한 롤백을 줄인다.
- 교착상태는 여러 트랜잭션이 서로 락 해제를 기다리는 상태이다.

시험 암기
- 동시성 제어 목적 : 무결성 유지 + 일관성 유지
- 동시성 제어 기법 : 락 / 타임스탬프 / 검증
- 락 종류 : 공유 락(S) / 배타 락(X)
- 양립성 : S-S 가능, S-X 불가, X-X 불가
- 2단계 락킹 : 요청 단계 → 반납 단계
- 반납 단계에서는 새로운 락 요청 불가
- 타임스탬프 : 작은 값의 트랜잭션이 먼저 수행
- 토마스 기록 규칙 : 오래된 쓰기 무시
- 교착상태 : 서로 기다리며 무한 대기

15강 회복 시스템
주요 용어
- 버퍼 블럭(Buffer Block) : 디스크에서 읽어와 주기억장치에 임시 저장된 블럭
- 로그 레코드(Log Record) : 데이터베이스의 모든 수정 작업을 기록한 데이터
- WAL(Write Ahead Logging) : 데이터 수정 전에 로그를 먼저 기록하는 회복 기법
- 체크포인트(Checkpoint) : 회복 비용을 줄이기 위해 수정된 내용을 주기적으로 디스크에 반영하는 기법

핵심 정리
- 데이터 회복(Recovery)은 장애 발생 시 데이터베이스를 장애 이전의 일관된 상태로 복구하는 작업이다.
- 실패 유형
  - 트랜잭션 실패 : 논리적 오류, 버퍼 오버플로 등
  - 시스템 장애 : 하드웨어 고장 등
  - 디스크 실패 : 디스크 손상
- 데이터는 디스크와 메모리 사이에서 블록 단위로 이동한다.
- 디스크의 블록은 물리적 블록, 메모리의 블록은 버퍼 블럭이라고 한다.
- 데이터 조작은 버퍼 블럭에서 수행되며, 트랜잭션 종료 또는 버퍼 공간 부족 시 디스크에 기록된다.
- 로그 기반 회복 기법은 데이터베이스의 모든 수정 작업을 로그에 기록하여 장애 발생 시 복구에 사용한다.
- WAL은 데이터를 수정하기 전에 로그를 먼저 안정 저장장치에 기록하는 방식이다.
- 회복 과정
  - Redo : 정상적으로 완료된 트랜잭션을 다시 실행
  - Undo : 완료되지 못한 트랜잭션을 이전 상태로 복구
- 장기간 운영 시 로그가 많아지므로 체크포인트를 사용하여 회복 시간을 줄인다.
- 체크포인트 작업
  - 모든 로그 레코드를 안정 저장장치에 기록
  - 수정된 모든 버퍼 블럭을 디스크에 반영
  - 현재 실행 중인 트랜잭션 목록 저장
- 체크포인트 이후에는 마지막 체크포인트 이후의 로그만 이용하여 Redo와 Undo를 수행하면 된다.

시험 암기
- 회복 : 장애 이전의 일관된 상태로 복구
- 실패 유형 : 트랜잭션 실패 / 시스템 장애 / 디스크 실패
- 버퍼 블럭 : 메모리에 있는 블럭
- 로그 레코드 : 모든 수정 작업 기록
- WAL : 데이터 수정 전 로그 먼저 기록
- Redo : 완료된 트랜잭션 재실행
- Undo : 미완료 트랜잭션 취소
- 체크포인트 목적 : 회복 시간 단축
- 체크포인트 이후 : 마지막 체크포인트 이후 로그만 확인
