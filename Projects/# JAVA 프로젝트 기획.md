## 1. 한정상품 구매 시스템

## 핵심 컨셉

판매자가 한정 수량 상품을 특정 시간에 오픈하면,
사용자는 출시 전부터 상품을 확인하고 알림을 신청한 뒤
오픈 시간에 대기열을 거쳐 선착순으로 구매하는 시스템

---

## 주요 사용자

### 판매자
- 상품 등록
- 출시 시간 예약
- 재고 수량 설정
- 1인 구매 제한 설정
- 판매 상태 관리

### 구매자
- 출시 예정 상품 조회
- 알림 신청
- 오픈 시간 대기
- 구매 요청
- 결제 대기
- 구매 완료

### 관리자
- 상품 승인
- 주문 현황 조회
- 실패 로그 조회
- 트래픽 현황 모니터링

---

## 추천 데이터 흐름

1. 판매자가 상품 등록
2. 출시 시간 예약
3. 사용자가 출시 예정 상품 확인
4. 사용자가 출시 알림 신청
5. 출시 시간이 되면 Scheduler가 상품 OPEN 처리
6. Redis에 상품 재고 세팅
7. 사용자들이 구매 요청
8. Rate Limit / 중복 요청 검사
9. Redis Queue 진입
10. Worker가 순차 처리
11. Redis 재고 차감
12. MySQL 주문 생성
13. 결제 대기 TTL 설정
14. 결제 완료 시 PAID 처리
15. 결제 만료 시 EXPIRED 처리 및 재고 복구

## 2 BidFlow: 실시간 경매 시스템

## 프로젝트 개요

사용자가 실시간으로 상품에 입찰하고, 최고가가 즉시 갱신되는 경매 플랫폼

동시 입찰 상황에서 가격 정합성을 보장하고, WebSocket을 통해 실시간 경매 현황을 제공하는 것이 목표

---

## 핵심 기능

* 경매 상품 등록
→ 상품명 / 시작가 / 종료 시간 등록
→ 경매 상태 관리

* 실시간 입찰
→ 사용자가 현재 최고가보다 높은 금액으로 입찰
→ 동시 입찰 시 최고가 정합성 보장
→ 중복/낮은 금액 입찰 차단

* 실시간 가격 갱신
→ WebSocket으로 최고가 즉시 반영
→ 참여자에게 입찰 현황 브로드캐스트

* 경매 종료 처리
→ 종료 시간 도달 시 자동 마감
→ 최종 낙찰자 선정
→ 낙찰 기록 저장

* 입찰 이력 관리
→ 상품별 입찰 내역 조회
→ 사용자별 입찰 내역 조회

* 알림 기능
→ 내가 최고가에서 밀렸을 때 알림
→ 경매 종료 알림
→ 낙찰 알림

---

## 자바/Spring 핵심 기술

### 동시성 처리
- 동일 상품에 여러 사용자가 동시에 입찰
- 낙관락 / 비관락 / Redis Lock 비교
- 최고가 정합성 보장

### 실시간 처리
- Spring WebSocket
- STOMP
- Redis Pub/Sub

### 트랜잭션
- 입찰 저장 + 최고가 갱신 원자성 보장
- 실패 시 Rollback 처리

### 스케줄링
- Scheduler로 경매 종료 처리
- 종료된 경매 낙찰자 확정

### 캐싱
- Redis에 현재 최고가 캐싱
- DB 조회 부하 감소

### 비동기 처리
- 입찰 알림 비동기 발송
- 입찰 로그 비동기 저장

---

## 성능 테스트
