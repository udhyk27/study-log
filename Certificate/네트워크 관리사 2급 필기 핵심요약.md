[네트워크관리사 2급 핵심 정리]

1. CSMA/CD

* Carrier Sense Multiple Access / Collision Detection
* 유선 Ethernet 방식
* 충돌 발생 후 감지하고 재전송
* 암기: 유선 = CD

2. CSMA/CA

* Collision Avoidance
* 무선 Wi-Fi 방식
* 충돌을 미리 회피
* 암기: 무선 = CA

3. Token Passing

* 토큰 가진 장치만 전송 가능
* 충돌 거의 없음
* Token Ring 사용

4. 클라우드 서비스

* SaaS = Software as a Service
  → 웹브라우저로 소프트웨어 제공
  → 일반 사용자 대상
  → 예: Google Docs, Microsoft 365

* PaaS = Platform as a Service
  → 개발 환경 제공
  → 개발자 대상

* IaaS = Infrastructure as a Service
  → 서버/네트워크/스토리지 제공

5. OSI 7계층

7 응용(Application)
→ 사용자 서비스

6 표현(Presentation)
→ 암호화 / 압축 / 인코딩

5 세션(Session)
→ 연결 생성 / 유지 / 종료

4 전송(Transport)
→ TCP/UDP, 흐름제어, 오류제어

3 네트워크(Network)
→ IP 주소, 라우팅

2 데이터링크(Data Link)
→ MAC 주소, 오류 검출

1 물리(Physical)
→ 전기 신호, 케이블

암기:
물데네전세표응

6. 장비 계층

* Hub → 1계층
* Switch → 2계층
* Router → 3계층
* Gateway → 7계층
* L3 Switch → 3계층

7. 데이터 단위(PDU)

* 1계층 → Bit
* 2계층 → Frame
* 3계층 → Packet
* 4계층 → Segment(TCP) / Datagram(UDP)

암기:
비프패세

8. TCP/IP 계층

* 응용계층
  → FTP, SMTP, TELNET, SNMP, DNS, MQTT

* 전송계층
  → TCP, UDP, RTCP

* 인터넷계층
  → IP, ICMP, IGMP, ARP, RARP

* 네트워크 액세스 계층
  → Ethernet, IEEE 802, HDLC, X.25, RS-232C

9. TCP / UDP

* TCP
  → 연결형(Connection-oriented)
  → 신뢰성 보장
  → 순서 보장
  → 흐름제어 / 오류제어 제공
  → 예: HTTP, HTTPS, FTP

* UDP
  → 비연결형(Connectionless)
  → 속도 빠름
  → 신뢰성 보장 X
  → 예: DNS, 스트리밍, 게임

암기:
신뢰성 = TCP
속도 = UDP

10. TCP 핵심 특징

* TCP는 프로세스 간 신뢰성 있는 통신 제공
* 오류 발생 세그먼트 재전송
* 순서 보장
* 흐름 제어 지원

11. 슬라이딩 윈도우

* TCP 흐름 제어 방식
* 수신측이 허용한 크기만큼 ACK 없이 전송 가능
* 네트워크 상황에 따라 전송량 동적 조절

쉽게:
"한 번에 얼마나 보내도 되는지 조절하는 기술"

12. 주요 프로토콜

* IP
  → 실제 패킷 전달
  → 비연결형

* ICMP
  → 오류 메시지 전달
  → ping 사용

* IGMP
  → 멀티캐스트 그룹 관리

* ARP
  → IP → MAC 주소 변환

* RARP
  → MAC → IP 주소 변환

13. 라우팅 핵심 용어

* Administrative Distance(관리 거리)
  → 라우팅 프로토콜 신뢰도
  → 값이 낮을수록 우선순위 높음

* Longest Match Rule
  → 목적지와 가장 길게 일치하는 경로 선택

* Next-hop Address
  → 다음 목적지 라우터 주소

* Metric
  → 경로 비용
  → 홉 수, 대역폭, 지연시간 등으로 계산

14. 자주 나오는 포트번호

* FTP → 20, 21
* SSH → 22
* TELNET → 23
* SMTP → 25
* DNS → 53
* HTTP → 80
* POP3 → 110

15. 전송 방식

* Unicast
  → 1 : 1 통신

* Broadcast
  → 1 : 전체 통신

* Multicast
  → 특정 그룹 통신

16. 흐름제어 / 오류제어 계층

* 전송계층(4계층)
  → 흐름제어
  → 오류제어
  → TCP 사용

* 데이터링크계층(2계층)
  → 오류 검출(CRC)

17. 서브넷 마스크 핵심

* IP 주소를 네트워크/호스트 영역 구분
* 네트워크 분할 시 사용

기본 클래스:

* A Class → 255.0.0.0
* B Class → 255.255.0.0
* C Class → 255.255.255.0

18. MAC(Media Access Control) 프로토콜

* 네트워크 공유 매체 접근 제어 방식
* 여러 장치가 동시에 통신할 때 충돌 방지/관리 목적

분류:

* 결정론적 방식
  → 접근 순서가 미리 정해짐
  → 충돌 거의 없음

* 확률론적 방식
  → 상황에 따라 접근
  → 충돌 가능성 존재

암기:
정해져 있으면 결정론적
랜덤이면 확률론적

19. MAC 프로토콜 종류

* Token Passing
  → 결정론적
  → 토큰 가진 장치만 전송
  → Ring 토폴로지 사용
  → 충돌 거의 없음

* Demand Priority(수요 우선순위)
  → 결정론적
  → 우선순위 높은 장치 먼저 전송
  → Bus / Star 토폴로지

* CSMA/CD
  → 확률론적
  → 유선 Ethernet
  → 충돌 발생 후 감지 후 재전송
  → Collision Detection

* CSMA/CA
  → 확률론적
  → 무선 Wi-Fi
  → 충돌 전 미리 회피
  → Collision Avoidance

암기:
유선 = CD
무선 = CA
토큰 = 링

20. 네트워크 토폴로지

* 네트워크 연결 구조(배치 방식)

종류:

* Bus(버스)
  → 한 선에 연결
  → 설치 저렴
  → 충돌 발생 가능

* Ring(링)
  → 원형 연결
  → Token Passing 사용

* Star(스타)
  → 중앙 장비(Switch/Hub) 중심
  → 가장 많이 사용

* Mesh(메시)
  → 장치끼리 다 연결
  → 장애 대응 강함
  → 비용 높음

* Tree(트리)
  → 계층 구조 연결

* Hybrid(하이브리드)
  → 여러 토폴로지 혼합

21. 물리적 / 논리적 토폴로지

* 물리적 토폴로지
  → 실제 케이블 연결 구조

* 논리적 토폴로지
  → 데이터 흐름 경로

중요:
물리적 구조와 논리적 구조는 다를 수 있음

22. 토폴로지 특징

* Bus
  → 설치 쉬움
  → 한 구간 문제 시 영향 큼

* Ring
  → 충돌 적음
  → 한 지점 장애 시 전체 영향 가능

* Star
  → 관리 쉬움
  → 중앙 장비 장애 시 전체 마비

* Mesh
  → 안정성 높음
  → 구축 비용 큼

23. 네트워크 선택 기준

* 네트워크 규모
* 장치 수
* 거리
* 대역폭 요구사항
* 장애 허용 수준(내결함성)

24. 내결함성(Fault Tolerance)

* 일부 장애 발생 시에도 정상 동작 능력
* Mesh 구조가 높음
* 시험 자주 출제됨

25. IPv4 vs IPv6 차이

* IPv4
  → 32bit 주소 체계
  → 클래스 기반(Classful) 주소 사용
  → 브로드캐스트 지원
  → 헤더 길이 가변

* IPv6
  → 128bit 주소 체계
  → 계층적 주소 구조 사용
  → 네트워크 접두사 + 인터페이스 식별자
  → 브로드캐스트 없음
  → 멀티캐스트 사용
  → 헤더 길이 고정
  → 보안(IPSec) 강화

암기:
IPv4 = 32bit
IPv6 = 128bit

26. IPv4 클래스(Classful)

* A Class
  → 대규모 네트워크
  → 기본 서브넷 마스크
  → 255.0.0.0

* B Class
  → 중규모 네트워크
  → 255.255.0.0

* C Class
  → 소규모 네트워크
  → 255.255.255.0

27. IPv6 특징

* 주소 공간 매우 큼
* 헤더 구조 단순화
* 처리 속도 향상
* 보안 강화
* 자동 주소 설정 지원
* 멀티캐스트 적극 사용

시험 포인트:
IPv6는 브로드캐스트 지원 X
→ 멀티캐스트 사용

28. 통신 방식

* Unicast
  → 1 : 1 통신

* Broadcast
  → 1 : 전체 통신
  → IPv4 지원
  → IPv6 미지원

* Multicast
  → 특정 그룹 통신
  → IPv6 핵심 방식
  → 효율적 그룹 통신 가능

쉽게:
Broadcast = 전체 알림
Multicast = 특정 방 초대

29. IPv6 멀티캐스트

* 특정 그룹 수신자에게만 데이터 전송
* 네트워크 트래픽 감소
* 불필요한 전송 감소
* 그룹 통신에 효율적

사용 예시:

* 멀티미디어 스트리밍
* 온라인 게임
* 화상회의
* IPTV

30. Active Directory 그룹 범위

* Global Group
  → 구성원: 자기 도메인 사용자만 가능
  → 권한 부여: 다른 도메인 포함 어디든 가능

쉽게:
"사람은 자기 동네만, 권한은 어디든"

* Domain Local Group
  → 구성원: 누구나 가능(다른 도메인 포함)
  → 권한 부여: 자기 도메인 리소스만 가능

쉽게:
"사람은 아무나, 권한은 우리 동네만"

* Universal Group
  → 구성원: 누구나 가능
  → 권한 부여: 어디든 가능

쉽게:
"사람도 자유, 권한도 자유"

암기:

구성원 제한

* Global → 자기 도메인만
* Domain Local → 누구나
* Universal → 누구나

리소스 접근 제한

* Global → 어디든 가능
* Domain Local → 자기 도메인 한정
* Universal → 어디든 가능

암기:
G = 사람 제한
DL = 권한 제한
U = 전부 자유

31. DNS 레코드 종류

* Host(A) 레코드
  → 호스트 이름 → IPv4 주소 매핑
  → 정방향 조회 영역(Forward Lookup Zone)

예:
```text
server.local → 192.168.0.10
```

시험 포인트:
"호스트 이름에 IP 연결"
→ A 레코드

* PTR(Pointer) 레코드
  → IP 주소 → 호스트 이름 변환
  → 역방향 조회 영역(Reverse Lookup Zone)

예:
```text
192.168.0.10 → server.local
```

시험 포인트:
"IP로 컴퓨터 이름 찾기"
→ PTR 레코드

암기:
A = 이름 → IP
PTR = IP → 이름

32. DNS 조회 영역

* 정방향 조회 영역
  → 도메인/호스트 이름으로 IP 조회

* 역방향 조회 영역
  → IP 주소로 호스트 이름 조회

암기:
정방향 = 이름 찾기
역방향 = 이름 역추적

33. DNS 주요 레코드

* SOA(Start of Authority)
  → DNS 영역 권한 정보
  → 기본 DNS 서버 정보 포함

* NS(Name Server)
  → 해당 DNS 영역의 이름 서버 정보

* A(Host)
  → 이름 → IPv4 주소

* PTR
  → IP 주소 → 이름

* CNAME
  → 별칭(Alias) 지정

* MX(Mail Exchange)
  → 메일 서버 지정

암기:
A = 주소
PTR = 역조회
MX = 메일
NS = 네임서버
SOA = 권한정보
