# 백엔드 포트폴리오 프로젝트 기획

---

# 1. 선착순 예약 시스템

## 프로젝트명

선착순 예약 관리 시스템

프론트: React
DB: MySQL
JPA

---

## 목표

* Java/Spring Boot 기반 실무형 백엔드 프로젝트
* 동시성, 트랜잭션, 락 처리 핵심 경험 확보
* 성능 테스트 및 수치 기반 개선 경험
* CI/CD, AWS 배포 포함

---

## 프로젝트 핵심

* 동일 자원(좌석/시간대) 선착순 예약
* 정원 제한
* 사용자 중복 예약 방지
* 동시 요청 충돌 상황 재현
* 데이터 정합성 보장

---

## 핵심 구현 기능

* 예약 생성 / 취소
* 사용자 중복 예약 방지
* 정원 초과 예약 방지
* 예약 상태 관리 (RESERVED, CANCELLED)
* 예약 성공/실패 로그 저장
* 관리자 예약 현황 조회 API

---

## 캐시 / 락

* Redis
  → 동시성 제어 핵심

---

## 테스트

* JUnit
  → 서비스 로직 검증
* 동시성 테스트
  → ExecutorService 활용

---

## 성능 테스트

* k6 또는 JMeter
  → 처리량, 실패율, 응답시간 측정

---

## 배포

* Docker
  → 실행 환경 통일
* GitHub Actions
  → push 시 자동 빌드/배포

---

## 클라우드

* AWS EC2
* AWS RDS
* AWS ElastiCache (Redis)

---

## 모니터링 / 로그

* 로그 저장 및 조회 API
* 장애 대응 흐름 정리

선택:

* Swagger
* JWT 인증
* 공통 예외 처리

---

## 자바/Spring 핵심 기술

### 객체지향 설계

* Controller / Service / Repository 분리
* 인터페이스 기반 설계
* 예약 정책 분리

### 동시성 처리

* synchronized
* ReentrantLock
* AtomicInteger
* ExecutorService 테스트

### 락 처리

* synchronized
* DB 비관락 (Pessimistic Lock)
* DB 낙관락 (Optimistic Lock)
* Redis 분산 락 (Redisson)

### 트랜잭션

* 예약 + 저장 원자성 보장
* 실패 시 롤백

### 재시도 / TTL

* Redis TTL 기반 만료 처리
* 재시도 로직으로 정합성 유지

### 비동기 처리

* @Async / CompletableFuture
* 로그 및 이벤트 분리

### 예외 처리

* 커스텀 예외
* 전역 예외 처리
* 공통 응답 구조

---

## 대규모 트래픽 처리

* Redis 재고 카운팅
* DB 부하 감소 구조
* 비동기 로그 처리
* 부하 테스트 기반 성능 측정

---

## 핵심 어필 포인트

* 동일 자원 동시 예약 충돌 상황 재현 및 해결
* 낙관락 / 비관락 / Redis 락 비교
* 재시도 로직 + TTL 기반 정합성 유지
* 트랜잭션 기반 데이터 정합성 보장
* 동시 요청 환경에서 재고 불일치 문제 해결
* 성능 테스트 기반 개선 수치화

DB: MySQL
(선택: Redis / Queue)

---

# 최종 포트폴리오 전략

## 역할 분리

* 예약 시스템 → 동시성, 락, 트랜잭션
* 로그 시스템 → 비동기, 데이터 처리, 성능 최적화

---

## 어필 방향

* 문제 재현 → 해결 → 성능 비교
* 수치 기반 결과 (TPS, 응답시간, 실패율)
* 실무 구조 기반 설계 경험 강조

---

## 최종 목표

동시성 제어 + 데이터 처리 구조를 모두 경험한
실무형 백엔드 개발 역량 포트폴리오 완성
