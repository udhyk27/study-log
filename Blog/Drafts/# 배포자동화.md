  ↓
빌드 + 테스트
  ↓
Spring Boot JAR 생성
  ↓
서버로 전송
  ↓
기존 앱 중지
  ↓

  ↓
Nginx가 요청을 받아 Spring으로 전달


1) 리포지토리 만들기
2) Pages 설정 켜기
3) 워크플로 파일 만들기
4) 다시 push
5) 주소 확인

[내 PC]
Spring Boot 프로젝트 개발

[GitHub]
소스코드 저장
main 브랜치 push

[GitHub Actions]
1. checkout
2. JDK 설치
3. 테스트
4. JAR 빌드
5. 서버 접속(SSH)
6. 서버에 JAR 복사
7. 서비스 재시작

[Linux Server]
/home/ubuntu/app/myapp.jar
/etc/systemd/system/myapp.service
Nginx reverse proxy

Client
  ↓
Nginx
  ↓
Spring Boot App
  ↓
DB(MySQL/MariaDB/PostgreSQL)

# 워크플로
name: Spring Boot CI/CD

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up JDK
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '17'
          cache: maven

      - name: Build
        run: ./mvnw clean package -DskipTests

      - name: Copy jar to server
        run: |
          echo "여기서 scp 또는 ssh 배포"

      - name: Restart app
        run: |
          echo "여기서 systemctl restart"
