암호화 알고리즘

보안을 이야기할 때 가장 기본이 되는 개념이 암호화입니다.
암호화는 크게 대칭키, 비대칭키, 해시 세 가지로 나누어 이해하시면 됩니다.

대칭키 암호화는 하나의 키로 데이터를 암호화하고 복호화하는 방식입니다. 대표적으로 AES가 있으며, 현재 가장 널리 사용됩니다. 과거에는 DES나 3DES도 사용되었지만, 보안 문제로 점점 사용이 줄어들고 있습니다. 이 방식은 속도가 빠르기 때문에 대용량 데이터 처리에 적합하지만, 같은 키를 공유해야 한다는 점에서 키 관리가 어렵습니다.

비대칭키 암호화는 공개키와 개인키, 두 개의 키를 사용하는 방식입니다. 대표적으로 RSA와 ECC가 있습니다. 공개키로 암호화한 데이터는 개인키로만 복호화할 수 있어 키 교환 과정에서 안전합니다. 다만 연산이 느리기 때문에 실제 데이터 암호화보다는 키 교환이나 인증에 주로 사용됩니다.

해시는 암호화와는 조금 다른 개념으로, 입력값을 일정한 길이의 값으로 변환하는 일방향 함수입니다. 대표적으로 SHA-256와 SHA-3가 사용되며, MD5나 SHA-1은 현재 취약점 때문에 사용이 권장되지 않습니다. 해시는 복호화가 불가능하기 때문에 비밀번호 저장에 적합합니다.

실무에서는 이 세 가지를 단독으로 사용하기보다는 조합해서 사용합니다. 예를 들어 HTTPS 통신에서는 RSA나 ECC로 키를 안전하게 교환하고, 실제 데이터는 AES로 암호화하는 구조를 사용합니다.

비밀번호와 Salt

비밀번호는 단순히 해시만 해서 저장하면 안전해 보이지만, 실제로는 그렇지 않습니다.

예를 들어 같은 비밀번호를 사용하는 사용자가 여러 명이라면, 해시 결과도 동일하게 나오게 됩니다. 이렇게 되면 공격자가 미리 만들어둔 해시 목록(레인보우 테이블)을 이용해 원래 비밀번호를 쉽게 추측할 수 있습니다.

이 문제를 해결하기 위해 사용하는 것이 salt입니다.

salt는 비밀번호에 임의의 값을 추가한 뒤 해시를 생성하는 방식입니다.
즉, 같은 비밀번호라도 salt 값이 다르면 결과 해시도 완전히 달라지게 됩니다.

예를 들어

password + A1 → 해시값1
password + B7 → 해시값2

처럼 결과가 달라지기 때문에, 공격자가 미리 계산해둔 값으로는 대응하기 어렵습니다.

실무에서는 보통 사용자마다 서로 다른 salt를 생성해서 함께 저장합니다. 그리고 로그인 시에는 동일한 salt를 붙여 다시 해시를 계산한 뒤 비교하는 방식으로 인증을 처리합니다.

또한 최근에는 단순 해시보다 더 안전한 방식으로 반복 연산을 추가한 알고리즘(예: bcrypt, PBKDF2 등)을 사용하여 공격 난이도를 높이는 경우가 많습니다.

사이버 공격과 보안 개념

암호화는 실제 공격을 막기 위한 수단이기 때문에, 어떤 공격이 있는지도 함께 알아두는 것이 중요합니다.

네트워크 구간에서 데이터를 가로채는 중간자 공격은 대표적인 사례입니다. 사용자의 통신을 중간에서 가로채 내용을 확인하거나 변조하는 방식이며, HTTPS와 TLS 암호화로 이를 방지할 수 있습니다.

서비스를 마비시키는 DDoS 공격도 자주 발생합니다. 다수의 요청을 동시에 보내 서버를 과부하시켜 정상적인 서비스를 어렵게 만드는 방식입니다.

웹 애플리케이션에서는 SQL Injection이나 XSS 같은 입력 기반 공격도 많습니다. 이는 암호화보다는 입력값 검증과 보안 코딩을 통해 방어하는 것이 중요합니다.

비밀번호 관련 공격으로는 브루트포스 공격이나 레인보우 테이블 공격이 있으며, salt와 해시, 그리고 반복 연산을 통해 방어합니다.

또한 암호화되지 않은 통신에서는 패킷 스니핑을 통해 정보가 그대로 노출될 수 있기 때문에, 반드시 HTTPS를 사용하는 것이 중요합니다.

정리

보안은 하나의 기술로 해결되는 것이 아니라 여러 요소가 함께 작동하면서 이루어집니다.

데이터는 AES로 보호하고, 키 교환은 RSA나 ECC로 처리하며, 비밀번호는 해시와 salt를 사용해 안전하게 저장합니다. 여기에 HTTPS 통신과 입력값 검증까지 더해져야 실제로 안전한 서비스가 됩니다.

결국 핵심은 단순합니다.
비밀번호는 해시와 salt로 보호하고, 통신은 HTTPS로 암호화하며, 입력값은 항상 검증하는 것이 기본적인 보안의 출발점입니다.
