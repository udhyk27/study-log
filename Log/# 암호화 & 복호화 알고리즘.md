# 암호화 알고리즘


🔐 1️⃣ 대칭키 암호화 (Symmetric)

👉 하나의 키로 암호화 + 복호화

대표 알고리즘
AES (가장 많이 씀, 사실상 표준)
DES (옛날, 지금은 거의 안 씀 ❌)
3DES (DES 개선판, 이것도 점점 사라지는 중)
특징
속도 빠름 ⚡
대용량 데이터 암호화에 적합
키 관리가 어려움 (같은 키 써야 해서)

👉 예: DB 암호화, 파일 암호화

🔑 2️⃣ 비대칭키 암호화 (Public Key)

👉 키가 2개 (공개키 + 개인키)

대표 알고리즘
RSA (가장 유명)
ECC (요즘 많이 쓰는 경량 방식)
특징
키 교환에 강함
속도 느림 🐢
작은 데이터 / 키 교환에 사용

👉 예: HTTPS, 인증서

🔒 3️⃣ 해시 (Hash) ← 암호화랑 조금 다름

👉 복호화가 없음 (일방향)

대표 알고리즘
SHA-256 (가장 많이 씀)
SHA-3
MD5 ❌ (깨짐)
SHA-1 ❌ (취약)
특징
비밀번호 저장에 사용
원래 값으로 되돌릴 수 없음

👉 예: 비밀번호 저장





# 실무에서 실제 사용 방식

👉 하나만 쓰는 게 아니라 조합해서 씀

예: HTTPS

RSA/ECC → 키 교환
AES → 실제 데이터 암호화

👉 이렇게 섞어서 사용

🎯 핵심 한 줄

👉 빠른 건 AES, 안전한 키 교환은 RSA/ECC, 비밀번호는 SHA-256

원하면
👉 “비밀번호 암호화 왜 해시 쓰는지 (salt 포함)”
👉 “JWT / HTTPS 내부 동작”
👉 “AES vs RSA 언제 선택하는지”
