자바 가비지 컬렉터(GC) 동작 원리 완벽 정리

자바에서는 개발자가 직접 메모리를 해제할 필요가 없습니다. 가비지 컬렉터가 더 이상 참조되지 않는 객체를 찾아 자동으로 메모리를 정리해주기 때문입니다. 그 핵심 구조와 과정을 살펴보겠습니다.
1. 힙(Heap) 메모리의 세대별 구조

자바는 효율적인 메모리 관리를 위해 힙 영역을 객체의 생존 기간에 따라 세대별로 나누어 관리합니다.

    Young Generation: 새롭게 생성된 객체들이 머무는 곳입니다. 이곳에서 발생하는 GC를 Minor GC라고 부르며, 빈번하게 일어납니다.

        Eden: 객체가 처음 생성되어 저장되는 영역입니다.

        Survivor 0 / 1: Eden에서 살아남은 객체들이 이동하는 중간 단계입니다.

    Old Generation: Young 영역에서 끝까지 살아남은 객체들이 이동하는 곳입니다. 이곳에서 발생하는 GC를 Major GC라고 하며, Young 영역보다 규모가 크고 시간이 오래 걸립니다.

2. GC의 기본 동작 과정 (Mark & Sweep)

가비지 컬렉션은 크게 두 가지 단계를 거쳐 실행됩니다.

    Mark (표시 단계): 메모리에서 사용 중인 객체와 그렇지 않은 객체를 식별하여 표시합니다.

    Sweep (제거 단계): 표시되지 않은(참조가 끊긴) 객체들을 메모리에서 삭제합니다.

    Compact (압축 단계): (필요 시) 제거 후 흩어진 객체들을 모아 메모리 단편화를 방지합니다.

3. Stop-The-World 현상

GC가 실행될 때는 GC를 담당하는 스레드를 제외한 모든 애플리케이션 스레드가 일시 정지됩니다. 이를 Stop-The-World라고 합니다. 이 중단 시간이 길어지면 서비스 성능에 악영향을 주므로, 이를 최소화하는 것이 GC 튜닝의 핵심입니다.
4. 주요 GC 알고리즘 종류

환경과 목적에 따라 다양한 GC 방식이 사용됩니다.

    Serial GC: 단일 스레드로 동작하며 메모리가 적은 환경에 적합합니다.

    Parallel GC: 멀티스레드를 사용하여 처리량을 높인 방식입니다.

    G1 GC (Garbage First): 대용량 메모리 환경에 최적화되어 있으며, 현재 많은 서버 환경에서 기본으로 사용되는 효율적인 방식입니다.

5. 전체 흐름 요약

    새 객체 생성 후 Eden 영역 저장

    Minor GC 발생 시 살아남은 객체는 Survivor 영역으로 이동

    계속해서 살아남은 객체는 Old Generation으로 승격

    Old 영역이 가득 차면 Major GC 발생 및 정리

    핵심 요약
    가비지 컬렉터는 참조가 끊긴 객체를 자동으로 수거하여 메모리 누수를 방지합니다. 효율적인 관리를 위해 힙을 Young과 Old 세대로 나누며, 실행 시 발생하는 Stop-The-World 시간을 줄이는 것이 성능 관리의 관건입니다.
